# AIコード作成サポート(VSCode活用 応用編)

## 1. このステップの目的

このステップでは、VSCodeのAI補助に加えて  
**チャット型AIを併用し、開発効率と品質を同時に高める方法**を学びます。

従来はエディタ内だけで完結していましたが、  
チャット型AIを組み合わせることで以下が可能になります：

* エラー原因の特定と理解
* コードの改善( リファクタリング )
* 設計レベルの相談

👉 **「書く」から「考えて改善する」へ進むことが目的です**

---

## 1.1 今回やること

以下のスキルを身につけます：

1. チャット型AIとエディタAIの役割分担
2. エラーをAIで解決する方法
3. コードの改善( リファクタリング )

---

## 2. AIの使い分け( 重要 )

### 2.1 役割の違い

| AIの種類 | 役割 |
|--------|------|
| エディタAI( Windsurfなど ) | コード補完・その場で書く |
| チャット型AI | 考える・調べる・改善する |

---

### 2.2 なぜ使い分けるのか

役割を分けることで、以下の効果があります：

* 無駄なやり取りが減る
* AIの精度が上がる
* 作業スピードが向上する

👉 **1つのAIに全部やらせると非効率になります**

---

### ✔ 重要ポイント

👉 **書く：エディタAI**  
👉 **考える：チャットAI**

---

## 3. エラーの解決方法

AIを使う最大の価値はここです。

### 3.1 エラーが出たらやること

👉 **エラー文をそのままコピー**

例：
```
ZeroDivisionError: division by zero
````

---

### 3.2 チャットAIに聞く

```text
このエラーの原因と解決方法を教えてください
````

---

### 3.3 より精度を上げる方法
※情報漏洩に注意、貼り付ける場合 エンタープライズAI かどうか確認が必要です

```text
このコードでエラーが出ます。原因と修正方法を教えてください。
{ ソースコードを貼り付ける }
```

---

### ✔ なぜこれが重要か

* エラー文だけ → 推測になる
* コードあり → 正確に特定できる

👉 **情報量が多いほど回答精度が上がる**

---

## 4. コードの改善( リファクタリング )
※情報漏洩に注意、貼り付ける場合 エンタープライズAI かどうか確認が必要です  
AIは「改善」に非常に強い領域です  

### 4.1 基本の聞き方

```text
このコードを改善してください
{ ソースコードを貼り付ける }
```

---

### 4.2 良い指示の例

```text
このコードを読みやすくしてください
{ ソースコードを貼り付ける }
```

```text
このコードをPythonらしい書き方にしてください
{ ソースコードを貼り付ける }
```

```text
処理を関数に分割してください
{ ソースコードを貼り付ける }
```

---

### 4.3 例( 改善前 )

```python
def average(values):
  total = 0
  for x in values:
    total += x
  return total / len(values)
```

---

### 4.4 改善後( 例 )

```python
def average(values):
  return sum(values) / len(values)
```

---

### ✔ なぜ改善が重要か

* バグが減る
* 可読性が上がる
* 保守しやすくなる

👉 **この考え方はコードだけでなく、ドキュメント改善にも有効です**  
　　(構造化・分かりやすさの向上に同じように効く)

---

## 5. AIに良い指示を出す方法
※情報漏洩に注意、貼り付ける場合 エンタープライズAI かどうか確認が必要です  
※参考：[AIリテラシー 6. プロンプトの基本](社員のスキルアップ教育計画_0101AIリテラシー.md#6-プロンプトの基本)

### 5.1 NG例

```text
直して
{ ソースコードを貼り付ける }
````

👉 情報不足で精度が低い

---

### 5.2 改善例①

```text
このコードのエラー原因を説明し、修正してください
{ ソースコードを貼り付ける }
```

👉 **Task(何をしてほしいか)** が明確になった状態

---

### 5.3 改善例②

```text
このコードのエラー原因を説明し、修正してください。
初心者でも分かるように解説を付けてください。
{ ソースコードを貼り付ける }
```

👉 **Constraint(制約)** が追加され、出力の質が上がる

---

### 5.4 改善例③

```text
あなたはPythonに詳しいエンジニアです。
このコードのエラー原因を説明し、修正してください。
初心者でも分かるように解説を付けてください。
{ ソースコードを貼り付ける }
```

👉 **Role(ロール)** を追加することで、回答の方向性が安定する

---

### ✔ ポイント

* 改善例①：Task(目的) → 原因と修正は出るが説明が不足する
* 改善例②：Task(目的) +Constraint(制約) → 理解しやすくなる
* 改善例③：Task(目的) +Constraint(制約) +Role(役割) → 説明の粒度や方向性が安定する

👉 **足りない情報を後から足すのが正しい使い方**

---

### ✔ なぜこれが重要か

* 曖昧 → AIが推測する → 間違える  
* 明確 → AIが判断できる → 正確になる  

👉 **AIは魔法ではなく「条件付き関数」**  
　　入力( 指示 )が変われば、出力( 結果 )も変わる

---

## 6. よくある失敗とその結果

### ❌ AIを信じすぎる

**なぜ起こるか：**
* 出力を検証せず、そのまま正しいと判断してしまうため

**何が起こるか：**
* 間違ったコードに気づかず、バグを埋め込む

---

### ❌ コピペだけする

**なぜ起こるか：**
* 内容を理解せずに使用しているため応用できない

**何が起こるか：**
* 使用する側が理解できず、こぴぺマシーンと化す

---

### ❌ 大きすぎるコードを投げる

**なぜ起こるか：**
* 問題を分割せずに丸投げしているため、意図が伝わらない

**何が起こるか：**
* 文脈が曖昧になり、回答精度が落ちる

---

### ✔ 回避方法

* 小さく分割して聞く
* 必ず内容を確認する
* 理由を理解する

---

## 7. チャット型AIの併用で重要なこと

### 7.1 AIは「相談相手」として使う

なぜ重要か：

* 人より早く回答が得られる
* 試行錯誤の回数を増やせる

---

### 7.2 エラーは学習のチャンス

なぜそうなるか：

* エラーには原因が明確にある
* 修正プロセスが理解につながる

---

### 7.3 最も価値が高いのは「改善」

なぜか：

* コード品質が上がる
* 実務では「書く」より「直す」時間が長い

---

### ✔ 初心者向けベストな使い方

1. エディタAIでコードを書く
2. エラーが出たらチャットAIで解決
3. 最後に必ず改善を依頼する

👉 **この3ステップを繰り返すのが最短成長ルートです**
