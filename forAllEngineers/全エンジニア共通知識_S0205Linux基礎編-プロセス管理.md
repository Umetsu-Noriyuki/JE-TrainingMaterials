# プロセスとは

Linuxでは、実行中のプログラムを「プロセス」と呼びます。

例えば、

- Webブラウザ
- エディタ
- Webサーバー
- データベース

なども実行されるとプロセスになります。

イメージとしては次のようになります。

```text
プログラム
↓
実行
↓
プロセス
```

例えば、

```text
nginx
```

というWebサーバープログラムを起動すると、

```text
nginxプロセス
```

としてLinux上で動作します。

---

# なぜプロセス管理が必要なのか

サーバーでは多くのプロセスが同時に動作しています。

例

```text
Linuxサーバー
├─ Webサーバー
├─ データベース
├─ ログ管理
└─ SSH接続
```

そのため、

- 現在何が動いているか
- 正常に動作しているか
- 不要なプロセスが存在しないか

を確認する必要があります。

---

# プロセスを確認する

## ps コマンド

現在実行中のプロセスを確認するには、

```bash
ps
```

を利用します。

実行例

```text
PID TTY          TIME CMD
1234 pts/0    00:00:00 bash
5678 pts/0    00:00:00 ps
```

表示される主な項目は次の通りです。

| 項目 | 説明 |
|--------|--------|
| PID | プロセスID |
| CMD | 実行中のプログラム名 |

---

## PIDとは

Linuxでは、実行中のプロセスごとに番号が割り当てられます。

この番号を

```text
PID（Process ID）
```

と呼びます。

例

```text
PID
1234
```

後ほど学習するプロセス停止などで利用します。

---

## ps aux

サーバー全体のプロセスを確認する場合は、

```bash
ps aux
```

を利用します。

実行例

```text
USER       PID %CPU %MEM COMMAND
root         1  0.0  0.1 systemd
user      1234  0.0  0.2 bash
```

主な項目は次の通りです。

| 項目 | 説明 |
|--------|--------|
| USER | プロセスを実行しているユーザー |
| PID | プロセスID |
| %CPU | CPU使用率 |
| %MEM | メモリ使用率 |
| COMMAND | 実行中のプログラム |

例えば、

```text
root         1  0.0  0.1 systemd
```

は、

- root ユーザーが実行している
- PIDは 1
- systemd というプログラム

を表しています。

また、

```text
user      1234  0.0  0.2 bash
```

は、

- user ユーザーが実行している
- PIDは 1234
- bash（シェル）が動作している

ことを表しています。

bash は、ターミナルで入力したコマンドをLinuxへ伝えるためのプログラムです。

このような役割を持つプログラムを「シェル」と呼びます。

シェルには複数の種類がありますが、bash は代表的なシェルの一つです。

まずは、

```text
どのユーザーが
どのプログラムを
どのPIDで実行しているか
```

を確認できるようになれば十分です。

多くのプロセスが表示されるため、サーバー運用ではよく利用されるコマンドです。

---

## top コマンド

現在のプロセスの状態をリアルタイムで確認するには、

```bash
top
```

を利用します。

```text
CPU使用率
メモリ使用量
実行中のプロセス
```

などを確認できます。

サーバーが重い場合の調査で利用することがあります。

---

# プロセスを停止する

Linuxでは、必要に応じて実行中のプロセスを停止することがあります。

例えば、

- 誤って起動したプログラムを終了したい場合
- フリーズして応答しなくなったプログラムを停止したい場合
- 異常動作しているプロセスを停止したい場合

などがあります。

サーバー運用では、動作しているプロセスを確認するだけでなく、必要に応じて停止することもあります。

ただし、重要なシステムプロセスを停止すると、システムやサービスが正常に動作しなくなる可能性があります。

そのため、停止するプロセスを十分に確認してから操作することが重要です。

---

## kill コマンド

プロセスを停止するには、

```bash
kill PID
```

を利用します。

例

```bash
kill 1234
```

この場合、

```text
PID 1234
```

のプロセスを停止します。

---

# サービスとは

Linuxでは、サーバーの起動中に継続して動作するプログラムを「サービス」として管理することがあります。

例えば、

```text
Webサーバー
データベース
SSHサーバー
```

などです。

イメージとしては、

```text
プログラム
↓
起動
↓
プロセス
↓
継続して動作
↓
サービス
```

となります。

---

# サービスを管理する

Linuxでは、サービスを管理するための仕組みが用意されています。

現在の多くのLinuxディストリビューションでは、

```text
systemd
```

という仕組みが利用されています。

systemd はサービスの起動や停止、状態管理などを行う役割を持っています。

イメージとしては次のようになります。

```text
systemd
↓
サービスを管理する仕組み
```

ただし、利用者が直接 systemd を操作することはあまりありません。

通常は、

```bash
systemctl
```

というコマンドを利用して systemd に指示を出します。

イメージとしては次のようになります。

```text
利用者
↓
systemctl
↓
systemd
↓
サービス
```

そのため、

- systemd → サービスを管理する仕組み
- systemctl → systemd を操作するコマンド

と理解しておけば十分です。

---

## サービス状態の確認

サービスの状態を確認するには、

```bash
systemctl status sshd
```

を利用します。

 ※sshd は SSH 接続を受け付けるサービスです。
SSHを利用すると、自分のPCからLinuxサーバーへ接続し、リモートでコマンド操作を行えます。
SSHについては後続の「SSH接続と実践演習」で詳しく学習します。

実行例

```text
active (running)
```

と表示されれば動作中です。

---

## サービスの起動

サービスを起動するには、

```bash
sudo systemctl start sshd
```

を利用します。

---

## サービスの停止

サービスを停止するには、

```bash
sudo systemctl stop sshd
```

を利用します。

---

## サービスの再起動

設定変更後などには、

```bash
sudo systemctl restart sshd
```

を利用します。

---

# プロセスとサービスの違い

初心者が混乱しやすいポイントとして、

```text
プロセス
サービス
```

があります。

違いを簡単にまとめると、

| 項目 | 説明 |
|--------|--------|
| プロセス | 実行中のプログラム |
| サービス | 継続的に動作するシステム機能 |

例えば、

```text
sshdサービス
↓
起動
↓
sshdプロセス
```

という関係になります。

# 用語集

| 用語 | 説明 |
|--------|--------|
| プロセス | 実行中のプログラム |
| PID | プロセスを識別するための番号（Process ID） |
| ps | プロセス一覧を表示するコマンド |
| ps aux | システム全体のプロセスを表示するコマンド |
| top | プロセスの状態をリアルタイムで確認するコマンド |
| kill | プロセスを停止するコマンド |
| bash | Linuxで広く利用されているシェルの一つ |
| シェル | ユーザーが入力したコマンドをLinuxへ伝えるプログラム |
| サービス | サーバー起動中に継続して動作するプログラムや機能 |
| systemd | Linuxでサービスを管理する仕組み |
| systemctl | systemdを操作してサービスを管理するコマンド |
| sshd | SSH接続を受け付けるサービス。SSHを利用すると、自分のPCからネットワーク経由でLinuxサーバーへ接続し、リモートからコマンド操作を行える |
