# Linux権限管理

## はじめに

前章では、Linuxのファイルやディレクトリの操作方法について学習しました。

本章では、Linuxのセキュリティの基礎となる「権限管理」について学習します。

Linuxでは複数のユーザーが同じコンピュータを利用することを前提としているため、

- 誰が利用するのか
- どのファイルを操作できるのか
- システム設定を変更できるのか

を管理する仕組みが備わっています。

この仕組みを理解することで、安全にLinuxサーバーを運用できるようになります。

---

## この章の目的

この章を学習することで、以下を理解できるようになることを目標とします。

- ユーザーとは何か説明できる
- グループの役割を理解できる
- 権限の基本的な仕組みを理解できる
- 管理者権限（root・sudo）の役割を理解できる

# なぜ権限管理が必要なのか

Linuxサーバーは複数の利用者が同時に利用することを前提としています。

例えば、

```text
サーバー
├─ 開発者
├─ 運用担当者
└─ システム管理者
```

のように複数人で利用することがあります。

もし権限管理が無ければ、

- 他人のファイルを削除する
- システム設定を変更する
- サービスを停止する

といった操作が誰でもできてしまいます。

そのためLinuxでは、

```text
誰が
何を
どこまで操作できるか
```

を管理する仕組みが用意されています。

---

# Linuxの権限管理の全体像

Linuxでは次のような流れでアクセス制御を行います。

```text
ユーザー
↓
グループに所属
↓
ファイルやディレクトリごとの権限が適用される
```

例えば、

```text
ユーザー
└─ user1
      ↓
グループ
└─ developers
```

という利用者がいたとします。

このとき、

```text
/opt/project
```

には

```text
読み取り可
書き込み可
```

が設定されている一方、

```text
/var/log
```

には

```text
読み取りのみ
```

が設定されている場合があります。

その場合、

```text
user1
↓
developers
↓
/opt/project
→ 読み取り可
→ 書き込み可

user1
↓
developers
↓
/var/log
→ 読み取り可
→ 書き込み不可
```

となります。

このようにLinuxでは、

```text
ユーザー
↓
グループ
↓
対象ごとの権限
```

によってアクセス制御が行われています。

---

# ユーザーとは

Linuxでは、コンピュータを利用する人やプログラムを「ユーザー」として管理します。

例えば、

```text
user1
user2
admin
```

などがユーザーです。

ログインすると、どのユーザーとして操作しているかがプロンプトに表示されます。

例

```text
user1@server:~$
```

この場合、

```text
user1
```

が現在ログインしているユーザーです。

---

## ユーザーごとのホームディレクトリ

ユーザーごとに専用の作業場所が用意されています。

例

```text
/home/user1
/home/user2
```

通常は自分のホームディレクトリ内で作業を行います。

---

# グループとは

グループとは、複数のユーザーをまとめて管理するための仕組みです。

例えば、

```text
developers
├─ user1
├─ user2
└─ user3
```

のように管理できます。

グループを利用することで、複数のユーザーに対してまとめて権限を設定できます。

---

## グループを利用する理由

例えば開発メンバーが利用する

```text
/opt/project
```

というディレクトリがあるとします。

開発メンバーが増減するたびに個別に権限を設定するのは大変です。

そこで、

```text
developers
```

というグループを作成し、

```text
developers
├─ user1
├─ user2
└─ user3
```

のように管理します。

これにより、グループ単位で権限を設定できるようになります。

---

# ユーザーとグループの関係

Linuxでは、ユーザーは必ず1つ以上のグループに所属します。

イメージとしては次のようになります。

```text
developers
├─ user1
├─ user2
└─ user3
```

Linuxでは、

```text
ユーザー
↓
グループに所属
↓
権限が適用される
```

という考え方になります。

実際のサーバー運用では、ユーザーごとに権限を設定するよりも、グループ単位で権限を管理することが一般的です。

---

# 権限とは

権限とは、

```text
ファイルやディレクトリに対して
どのような操作を許可するか
```

を制御する仕組みです。

---

## 権限の種類

Linuxの権限には主に3種類あります。

| 権限 | 意味 |
|--------|--------|
| r | 読み取り（Read） |
| w | 書き込み（Write） |
| x | 実行（Execute） |

---

## ファイルの権限

例えば、

```text
-rw-r--r--
```

という表示があります。

これは

```bash
ls -l
```

コマンドで確認できます。

ファイルの場合は、

| 権限 | 意味 |
|--------|--------|
| r | ファイルの内容を読める |
| w | ファイルに書き込める・ファイルを削除できる |
| x | ファイルを実行できる（実行形式ファイルの場合） |

という意味になります。

---

## ディレクトリの権限

ディレクトリにも権限があります。

例えば、

```text
drwxr-xr-x
```

という表示があります。

ディレクトリの場合は、

| 権限 | 意味 |
|--------|--------|
| r | 一覧を表示できる |
| w | ファイルやディレクトリを作成・削除できる |
| x | ディレクトリ内へ移動できる |

となります。

---

# 所有者という考え方

Linuxでは、ファイルやディレクトリごとに「誰が管理するものか」を設定できます。

この管理するユーザーのことを「所有者（Owner）」と呼びます。

例えば、user1 が作成したファイルは、通常 user1 が所有者になります。

```text
user1 が作成
↓
test.txt
↓
所有者は user1
```

そのためLinuxでは、

```text
誰のファイルなのか
```

を管理するために所有者という仕組みが用意されています。

通常は、ファイルやディレクトリを作成したユーザーが所有者になります。

## ls -l で確認してみる

ファイルやディレクトリの所有者や権限は、

```bash
ls -l
```

コマンドで確認できます。

例えば、

```text
-rw-r--r-- 1 user1 developers 100 test.txt
```

という表示があった場合、次のような意味になります。

```text
-rw-r--r-- 1 user1 developers 100 test.txt
││         │ │     │          │   │
││         │ │     │          │   └ ファイル名
││         │ │     │          └ ファイルサイズ
││         │ │     └ グループ
││         │ └ 所有者
││         └ その他の情報
│└ 権限
└ 形式(-: ファイル、d: ディレクトリ、l: シンボリックリンク)
```

この章では特に次の3つを理解しておきましょう。

| 項目 | 内容 |
|--------|--------|
| rw-r--r-- | このファイルに設定されている権限<br>- 左側3文字: 作成したユーザーの権限<br>- 中央3文字: 作成したユーザーが属するグループの権限<br>- 右側3文字: その他ユーザーの権限 |
| user1 | このファイルの所有者 |
| developers | このファイルに設定されているグループ |

ファイルサイズやその他の情報については、今後の学習で必要に応じて扱います。

---

# 管理者権限とは

ここまで学習したように、Linuxではユーザーごとに利用できる機能が制限されています。

例えば一般ユーザーは、

- 自分のファイルを操作する
- アプリケーションを利用する

といった通常の作業を行います。

一方で、

- ユーザーを作成する
- ソフトウェアをインストールする
- システム設定を変更する
- サービスを起動・停止する

といったシステム全体に影響する操作もあります。

これらの操作は、誰でも実行できるようにするとシステムの安全性が損なわれるため、特別な権限を持つユーザーだけが実行できるようになっています。

この特別な権限を「管理者権限」と呼びます。

Linuxでは、管理者権限を持つ特別なユーザーを

```text
root
```

と呼びます。

---

## rootユーザー

rootはLinuxシステム全体を管理できる特別なユーザーです。

管理者権限については前のセクションで説明した通り、システム全体に影響する操作を実行できます。

Linuxでは、通常の利用者とは区別された特別なユーザーとして root が用意されています。

例

```text
root@server:~#
```

一般ユーザーとの違いは、プロンプトの末尾が

```text
#
```

で表示されることです。

一方、一般ユーザーの場合は

```text
user1@server:~$
```

のように

```text
$
```

で表示されます。

実際の運用では、常に root で作業するのではなく、通常は一般ユーザーでログインし、必要な場合のみ管理者権限を利用することが一般的です。

---

## sudoとは

通常は一般ユーザーで作業し、

管理者権限が必要な場合だけ sudo を利用します。

例

```bash
sudo dnf install nginx
```

このコマンドは、Webサーバーソフトウェアである Nginx をインストールする例です。

sudo を付けることで、一時的に管理者権限で実行できます。

---

## なぜsudoを利用するのか

常にrootで作業すると、

- 誤って重要なファイルを削除する
- システム設定を変更してしまう

といったリスクがあります。

そのため実際の運用では、

```text
通常作業
↓
一般ユーザー

管理作業
↓
sudo
```

という使い方が一般的です。

---

# まとめ

- Linuxではユーザーごとに管理される
- ユーザーは1つ以上のグループに所属する
- グループを利用して複数ユーザーをまとめて管理できる
- 権限には r・w・x がある
- ファイルやディレクトリには所有者が存在する
- 権限は所有者・グループ・その他に対して設定される
- rootは管理者ユーザーである
- sudoを利用すると一時的に管理者権限で実行できる

# 用語集

| 用語 | 説明 |
|--------|--------|
| ユーザー | Linuxを利用する人やプログラムを識別する単位 |
| グループ | 複数のユーザーをまとめて管理する仕組み |
| 権限 | ファイルやディレクトリに対して許可された操作 |
| 所有者（Owner） | ファイルやディレクトリを所有するユーザー |
| r | 読み取り権限（Read） |
| w | 書き込み権限（Write） |
| x | 実行権限（Execute） |
| root | Linuxの管理者ユーザー |
| sudo | 一時的に管理者権限でコマンドを実行する仕組み |
| 管理者権限 | システム全体を変更できる特別な権限 |
| ホームディレクトリ | ユーザーごとの作業用ディレクトリ |
| 所有者 | ファイルやディレクトリを作成・管理するユーザー |
| グループ権限 | グループに所属するユーザーへ適用される権限 |
| その他（Others） | 所有者・グループ以外のユーザー |
