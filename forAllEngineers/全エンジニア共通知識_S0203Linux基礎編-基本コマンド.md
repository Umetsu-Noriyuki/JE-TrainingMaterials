# ターミナルとは

Linuxでは、コマンドを入力して操作を行うことがよくあります。

コマンドを入力するための画面を**ターミナル**と呼びます。

イメージとしては、Windowsの「コマンドプロンプト」や「PowerShell」に近いものです。

```text
+-----------------------------------+
| user@server:~$                    |
|                                   |
| ls                                |
| Documents Downloads test.txt      |
|                                   |
+-----------------------------------+
```

この画面にコマンドを入力してLinuxを操作します。

---

## プロンプトとは

ターミナルを開くと、次のような表示が出ます。

```text
user@server:~$
```

この表示を**プロンプト**と呼びます。

プロンプトには現在の利用者や接続先などの情報が表示されています。

それぞれの意味は以下の通りです。

```text
user@server:~$
│    │      │
│    │      └ 現在の場所
│    └ 接続先のコンピュータ名
└ ログイン中のユーザー名
```

| 表示 | 意味 |
|--------|--------|
| user | ユーザー名 |
| server | コンピュータ名（ホスト名） |
| ~ | ホームディレクトリ |
| $ | 一般ユーザーであることを示す記号 |

---

## ホームディレクトリとは

ホームディレクトリとは、ユーザーごとに用意された作業場所です。

例えば、ユーザー名が `user` の場合、

```text
/home/user
```

がホームディレクトリになります。

プロンプトではホームディレクトリを省略して

```text
~
```

で表示することがあります。

そのため、

```text
user@server:~$
```

は、

```text
ユーザー user が
server というコンピュータにログインしていて
現在は /home/user にいる
```

という意味になります。

Linuxでは現在作業している場所を「カレントディレクトリ」と呼びます。

現在のディレクトリを確認したり移動したりする方法は、本章の後半で学習します。

---

# コマンドの実行

コマンドとは、Linuxへ指示を出すための命令です。

例えば、

```bash
ls
```

というコマンドを実行すると、現在のディレクトリにあるファイルやディレクトリの一覧を表示できます。

実行例

```text
Documents
Downloads
Pictures
```

コマンドを入力した後に Enter キーを押すことで実行されます。

---

## コマンドの基本構造

多くのコマンドは次のような構造になっています。

```text
コマンド オプション 引数
```

例

```bash
ls -l /home
```

| 要素 | 説明 |
|--------|--------|
| ls | コマンド |
| -l | オプション |
| /home | 引数 |

---

## オプションとは

オプションはコマンドの動作を変更するための指定です。

例えば、

```bash
ls
```

はファイル一覧を表示します。

```bash
ls -l
```

とすると、詳細情報付きで表示されます。

---

## 引数とは

引数はコマンドの対象を指定するための値です。

例えば、

```bash
cd /home
```

では、

- cd がコマンド
- /home が引数

になります。

---

# 現在のディレクトリを確認する

現在作業している場所（ディレクトリ）を確認するには `pwd` コマンドを利用します。

```bash
pwd
```

実行例

```text
/home/user
```

pwd は以下の略です。

```text
Print Working Directory
```

「現在の作業ディレクトリを表示する」という意味になります。

---

# ファイル一覧を確認する

現在のディレクトリに存在するファイルやディレクトリを確認するには `ls` コマンドを利用します。

```bash
ls
```

実行例

```text
Documents
Downloads
Pictures
```

---

## 詳細情報を表示する

```bash
ls -l
```

実行例

```text
drwxr-xr-x 2 user user 4096 Jan 1 10:00 Documents
-rw-r--r-- 1 user user 100 Jan 1 10:00 test.txt
```

詳細表示については後続の「ユーザーと権限管理」で学習します。

---

# ディレクトリを移動する

Linuxではフォルダのことを「ディレクトリ」と呼びます。

ディレクトリを移動するには `cd` コマンドを利用します。

## ホームディレクトリへ移動

```bash
cd
```

または

```bash
cd ~
```

## 指定したディレクトリへ移動

```bash
cd /home
```

## 一つ上のディレクトリへ移動

```bash
cd ..
```

例

```text
/home/user/documents
        ↓
cd ..
        ↓
/home/user
```

---

# パスとは

Linuxではファイルやディレクトリの場所を表す文字列を**パス**と呼びます。

例

```text
/home/user/test.txt
```

これは

```text
/
└─ home
   └─ user
      └─ test.txt
```

を意味します。

---

## Windowsとのパス表記の違い

Linuxのパスは、Windowsとは表記方法が異なります。

### ディレクトリの区切り文字

Windows

```text
C:¥Users¥user¥Documents
```

Linux

```text
/home/user/documents
```

Linuxではディレクトリの区切りに `/`（スラッシュ）を使用します。

Windowsで利用する `¥` は使用しません。

---

### ドライブ名が存在しない

Windowsでは、

```text
C:
D:
E:
```

のようなドライブ名があります。

例えば、

```text
C:¥Users¥user
```

のようにファイルの場所を表します。

一方、Linuxにはドライブ名の概念がありません。

Linuxではすべてのファイルやディレクトリが

```text
/
```

（ルートディレクトリ）

を起点とした一つの階層構造で管理されています。

```text
/
├─ home
├─ etc
├─ var
└─ usr
```

そのため、

```text
/home/user
```

のような形式で場所を表します。

---

### 初学者が間違えやすい例

Windowsの感覚で次のように入力すると正しく動作しません。

```bash
cd C:¥Users¥user
```

Linuxでは次のように入力します。

```bash
cd /home/user
```

> ※ Linuxや技術文書では「¥」ではなく「\」（バックスラッシュ）として表記されることがあります。日本語環境では同じ文字として表示される場合があります。

---

## 絶対パス

ルートディレクトリ（ `/` ）から始まるパスです。

例

```text
/home/user/test.txt
```

どこにいても同じ場所を指します。

---

## 相対パス

現在のディレクトリを基準にしたパスです。

例

```text
documents/test.txt
```

現在の場所によって指す場所が変わります。

---

# コマンドのヘルプを確認する

## manコマンド

Linuxにはマニュアルが用意されています。

```bash
man ls
```

これにより `ls` コマンドの説明を確認できます。

---

## --help オプション

多くのコマンドでは `--help` を利用できます。

```bash
ls --help
```

実行例

```text
Usage: ls [OPTION]... [FILE]...
```

---

# まとめ

- Linuxはターミナルから操作することが多い
- プロンプトにはユーザー名や現在の場所が表示される
- `~` はホームディレクトリを表す
- `pwd` で現在の場所を確認できる
- `ls` でファイル一覧を確認できる
- `cd` でディレクトリを移動できる
- Linuxでは `/` を使用する
- Linuxには `C:` のようなドライブ名がない
- `man` や `--help` で使い方を調べられる
