# IaCトラブルシュート<br>( CREATE_FAILED、ROLLBACK、IAM、Dependency、Drift )

## 1. IaCトラブルシュートとは？

CloudFormation は便利ですが、  
実務では **作成失敗・更新失敗・削除失敗** が発生することがあります。

例えば：

- Stack 作成が途中で止まる
- 更新時にロールバックされる
- 削除できない
- 権限不足で失敗する
- 手作業変更で差異が出る

👉 エラー時に落ち着いて原因を確認する力が重要です。

---

## 2. まず確認する場所

トラブル時は CloudFormation の **Events** を確認します。

```plaintext
CloudFormation
 ↓
対象 Stack
 ↓
Events
````

確認ポイント：

* 失敗したリソース名
* エラーメッセージ
* どの処理で止まったか

👉 最初に Events を見る習慣が重要です。

---

## 3. CREATE_FAILED

作成時に失敗した状態です。

### よくある原因

* リソース名重複
* IAM 権限不足
* 必須パラメータ不足
* YAML記述ミス

### 例

```plaintext
CREATE_FAILED
Bucket already exists
```

👉 既存名と重複しないよう修正します。

---

## 4. UPDATE_ROLLBACK_COMPLETE

更新失敗後、自動的に元へ戻された状態です。

```plaintext
変更適用
 ↓
失敗
 ↓
自動ロールバック
```

### 主な原因

* 変更内容の不整合
* 削除できない依存関係
* 一部サービス制約

👉 変更差分を見直して再実行します。

---

## 5. IAM権限不足

CloudFormation 実行者または作成リソース側の権限不足です。

### 例

```plaintext
AccessDenied
Not authorized
```

### 確認点

* 実行ユーザー権限
* CloudFormation 実行Role
* 作成対象サービス権限

👉 IAM不足は非常に多い失敗原因です。

---

## 6. 削除できないケース

### よくある例

* S3 バケット内にファイルが残っている
* ENI が残っている
* Security Group が参照中
* RDS 削除保護が有効

```plaintext
delete-stack
 ↓
DELETE_FAILED
```

👉 依存先を解除してから再実行します。

---

## 7. Drift( 手作業変更差異 )

テンプレート定義と実環境がズレた状態です。

### 例

```plaintext
テンプレート:
EC2 t3.micro

実環境:
手作業で t3.large
```

### 問題点

* 再デプロイで戻る
* 原因不明障害になる
* 管理不能になる

👉 AWS Console 直接変更を避け、コード修正を基本にします。

---

## 8. 調査の基本手順

```plaintext
① Events確認
② 失敗リソース確認
③ エラー文確認
④ IAM / 依存関係確認
⑤ テンプレート修正
⑥ 再実行
```

👉 感覚ではなく、順番に確認することが重要です。

---

## 9. まとめ

* まず Events を確認する
* CREATE_FAILED は原因特定しやすい
* IAM不足は頻出
* 削除失敗は依存関係を疑う
* 手作業変更は Drift の原因

👉 IaC は「作る力」と同じくらい「直す力」が重要です。

---

## 10. 重要用語説明

| 用語                       | 説明                 |
| ------------------------ | ------------------ |
| CREATE_FAILED            | 作成処理が失敗した状態。       |
| UPDATE_ROLLBACK_COMPLETE | 更新失敗後、元へ戻された状態。    |
| DELETE_FAILED            | 削除処理が失敗した状態。       |
| Events                   | Stack実行履歴・エラー確認画面。 |
| AccessDenied             | 権限不足エラー。           |
| Dependency               | 他リソースとの依存関係。       |
| Drift                    | コード定義と実環境差異。       |
| Rollback                 | 失敗時に元へ戻す処理。        |
