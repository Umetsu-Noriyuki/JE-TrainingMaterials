# IaCイベント通知構築<br>( EventBridge、SNS、Rule、Permission )

## 1. イベント通知構成とは？

イベント通知構成とは、  
**何かが起きたことをきっかけに、自動で処理や通知を行う仕組み**です。

例：

- 毎朝9時に定期バッチ起動
- ファイル保存時に通知送信
- 障害発生時に管理者へ連絡
- AWSイベント発生時に自動処理

AWSでは主に以下を利用します。

- **EventBridge**：イベント検知・スケジュール起動
- **SNS**：通知配信
- **Lambda**：自動処理
- **IAM Permission**：実行許可

```plaintext
イベント発生
   ↓
EventBridge
   ↓
Lambda / SNS
```

👉 手動対応を減らし、業務自動化しやすくなります。

## 2. IaCで構築する意義

管理画面から設定すると、後から構成把握が難しくなります。

IaC化すると、

* 起動条件をコードで管理できる
* 通知先変更をレビューできる
* 同じ構成を再現できる
* Gitで変更履歴を残せる
* 環境ごとに分離しやすい

```plaintext id="5z8j2q"
手動設定
誰が何を設定したか不明

IaC設定
条件・通知先がコードで明確
```

## 3. EventBridge とは？

EventBridge はイベントを受け取り、
別サービスへ連携する仕組みです。

### 主な用途

| 用途        | 例         |
| --------- | --------- |
| 定時起動      | 毎日9時実行    |
| AWSイベント連携 | EC2停止通知   |
| 業務イベント連携  | ファイル到着時処理 |

```plaintext id="t7r2v9"
イベント
 ↓
EventBridge
 ↓
Lambda / SNS / SQS
```

## 4. Rule( 起動条件定義 )

EventBridge では Rule で条件を定義します。

### 定時実行例

```yaml id="j6p1qn"
Resources:
  DailyRule:
    Type: AWS::Events::Rule
    Properties:
      ScheduleExpression: rate(1 day)
```

### cron例

```plaintext
cron(0 9 * * ? *)
毎日 9:00 実行
```

### イベント検知例

* EC2起動時
* S3保存時
* CodeBuild失敗時

👉 Rule が「いつ・何を契機に動くか」を決めます。

## 5. SNS( 通知サービス )

SNS はメッセージを複数宛先へ通知できます。

例：

* メール通知
* Lambda起動
* 他システム連携

```plaintext id="y6lf4w"
障害発生
 ↓
SNS
 ├─ メール通知
 ├─ Lambda起動
 └─ 他システム連携
```

### IaC定義例

```yaml id="4s4d0o"
Resources:
  AlertTopic:
    Type: AWS::SNS::Topic
```

## 6. Permission( 実行許可 )

AWSサービス同士の連携には許可設定が必要です。

例：

* EventBridge → Lambda実行
* SNS → Lambda実行

```plaintext id="7u13ak"
EventBridge
   ↓ 許可必要
Lambda
```

### Lambda許可例

```yaml id="5v8d3m"
Resources:
  InvokePermission:
    Type: AWS::Lambda::Permission
```

👉 作成しても Permission不足で動かないケースは多いです。

## 9. よくある失敗例

| 失敗          | 原因           |
| ----------- | ------------ |
| 定時起動しない     | cron設定ミス     |
| Lambda呼ばれない | Permission不足 |
| 通知来ない       | SNS購読未確認     |
| dev/prod誤通知 | Topic分離不足    |

## 10. まとめ

* EventBridge はイベント連携の中心
* Rule で起動条件を定義する
* SNS は通知配信担当
* Permission は実行許可設定
* IaC化で条件・通知先を安全管理できる

👉 イベント通知構成は **業務自動化・障害検知・定時処理** に強い仕組みです。

## 11. 重要用語説明

| 用語 | 説明 |
|---|---|
| EventBridge( イベント連携 )<br>Amazon EventBridge | イベント発生や定時実行を起点に処理を連携するサービス。 |
| Rule( 実行条件 )<br>Rule | EventBridgeでイベント条件を定義する設定。 |
| SNS( 通知サービス )<br>Amazon Simple Notification Service | メール等へ通知配信できるサービス。 |
| Permission( 実行許可 )<br>Permission | AWSサービス間の実行許可設定。 |
| cron( 定時実行式 )<br>cron expression | 時刻や周期を指定する定時実行書式。 |
| Lambda( 関数実行サービス )<br>AWS Lambda | サーバー管理不要でプログラムを実行できるサービス。 |
| Deploy( 反映 )<br>Deploy | コードや設定をAWS環境へ適用すること。 |
| Event Driven( イベント駆動 )<br>Event Driven Architecture | イベント発生を起点に処理する設計方式。 |
| Schedule( 定時実行 )<br>Schedule | 指定時刻や周期で自動実行する仕組み。 |
| Notification( 通知 )<br>Notification | 利用者や管理者へ知らせる処理。 |
| Topic( 通知先グループ )<br>Topic | SNSで配信先をまとめる単位。 |
| Subscription( 購読設定 )<br>Subscription | SNS通知の受信先登録設定。 |
