# 管理ツールの活用法( Management Tools )
AWSを安全かつ効率的に管理・運用するための基本ツールと権限設計の基礎を理解し、  
セキュリティ強化のポイントまで押さえて、運用の土台を固めます。

## 1. Management Console( 管理ツール ) - 操作の起点を理解する
AWS Management Consoleは、AWSをブラウザから操作する入口です。  
AWSの各種サービスをブラウザ上で操作できる統合管理ツールで、  
初心者でも直感的にリソースを管理できる画面が提供されています。  

ここから各サービスの設定画面へ移動します。  

- 代表的なサービスへアクセス可能( EC2、S3、IAMなど )  
- GUIによるリソースの作成、設定、監視が簡単  
- 権限によって表示される項目が変わり、操作範囲が制限される  

実務では、運用担当者ごとにアクセスを制限し、不要な操作からシステムを守ります。

## 2. Region( リージョン ) - 地理的区分を理解し最適化する
AWSは世界中に約30拠点のデータセンターを設置しており、  
Regionはその所在地を示す地理的な区分です。  

サービスはこのRegion単位で分かれており、
同じリージョン内での通信は高速かつ低遅延です。

各Regionには複数のAZ( Availability Zone )があり、  
AZは独立したデータセンター群として障害対策や可用性向上に利用されます。

| ポイント | 説明 |
| -------- | ---- |
| Regionの例 | 東京 ap-northeast-1、米国バージニア us-east-1など |
| 利用目的 | サービスの可用性確保や法令遵守( データの所在管理 )で重要 |
| 選定基準 | ユーザーの所在地、コスト、レイテンシ、データ保護ルール |

複数Regionを跨ぐ構成では、データのレプリケーションやバックアップ戦略も考慮します。

## 3. IAM( Identity and Access Management ) - 認証・権限管理の基礎
IAMはAWSの認証と権限の管理を担うサービスで、  
誰がどの操作をできるか厳密に制御することが可能です。  
運用のセキュリティと効率を大きく左右する非常に重要な要素です。

### 3-1. IAMの基本構成要素

| 要素名 | 説明 |
|--------|------|
| ユーザー | AWSリソースにアクセスする人やシステム |
| グループ | 複数ユーザーをまとめて権限管理 |
| ロール | 一時的に割り当てる権限セット( サービス間や組織間連携用 ) |
| ポリシー | 許可・拒否のルール( JSON形式 ) |

### 3-2. 権限設計の基礎 - 最小権限の原則
不正利用・情報漏えい・誤操作による被害範囲を最小化するため、重要です。
- 必要な操作だけに限定して権限を付与する( 最小権限 )  
- 役割ごとにポリシーを作成し、グループかロールに割り当てる  
- 長期間使わないアクセスキーは無効化・削除する  
- IAMポリシーは定期的に見直し、過剰な権限を防止する  

### 3-3. 実務でのベストプラクティス

- **管理者権限アカウントは限定し、MFA( 多要素認証 )を必ず設定**  
  → 乗っ取り時の被害防止と重要操作の保護
- **サービス連携にはロールを使い、一時的な権限を利用**  
  → 長期認証情報を減らし、漏えいリスクを低減
- **ログイン監査や操作履歴はCloudTrailで記録・監視**  
  → 問題発生時の追跡・内部統制・監査対応
- **権限は役割変更に伴って速やかに更新・削除**  
  → 不要権限の残存防止

### 3-4.【図解】AWS管理の基本構造イメージ

```plaintext
ユーザー
   ├── IAM ユーザー( 個人アカウント )
   ├── IAM グループ( 例：開発者、運用 )
   └── IAM ロール( サービス間連携や一時的権限 )
        ↓
  ポリシーが権限を決定
        ↓
  AWSリソースにアクセス( EC2, S3, RDS等 )
```

## 4. 安全にAWSを使うための設定ポイント
IAM や CloudTrail を適切に設定することで、AWS環境の安全性向上と、運用効率化につながります。  

### IAM 側で行う主な設定
- 権限は最小限にし、不要権限は削除  
  → 誤操作や不正利用時の被害範囲を最小化  
  → チームごとの適切なアクセス権管理が可能
- 多要素認証( MFA )の導入を推奨  
  → パスワード漏えい時の不正ログイン防止  
  → 管理者アカウントの安全性向上
- 管理者アカウントは限定的に利用  
  → 重要設定の変更リスクを低減  
  → 内部統制や監査対応を強化
- 定期的にポリシーを見直し、不整合を排除  
  → 不要権限の残存防止  
  → 異動・退職時の権限整理を迅速化

### CloudTrail 側で行う主な設定
- 操作ログ( CloudTrail )で監査を実施  
  → 監査証跡を確保  
  → 問題発生時の原因調査や追跡が可能  
  → 法規制遵守への対応を支援

## 5. 重要用語説明
| キーワード | 説明 |
|---|---|
| Infrastructure( インフラストラクチャ ) | ITシステムの基盤。<br>サーバーやネットワークなどのハード・ソフトの総称 |
| AWS<br>Amazon Web Services | Amazonが提供するクラウドサービスの総称 |
| Management Console( 管理コンソール ) | AWSのGUI管理画面 |
| Region( リージョン ) | AWSデータセンターの地理的区分 |
| AZ<br>Availability Zone | Region内にある独立したデータセンター群。障害対策や高可用性に利用される |
| IAM<br>Identity and Access Management | AWSの認証・権限管理サービス |
| User( ユーザー ) | AWSへログインして操作する個人アカウント |
| Group( グループ ) | 複数ユーザーをまとめて同じ権限を付与する仕組み |
| Role( ロール ) | 一時的に付与される権限セット |
| Policy( ポリシー ) | 認可ルール( 許可・拒否 )の定義 |
| MFA<br>Multi-Factor Authentication | ID・パスワードに加えて追加認証を行う多要素認証 |
| Access Key( アクセスキー ) | CLIやプログラムからAWSへ接続するための認証情報。厳重な管理が必要。 |
| CloudTrail | AWS上の操作履歴やAPI実行履歴を記録する監査サービス |
