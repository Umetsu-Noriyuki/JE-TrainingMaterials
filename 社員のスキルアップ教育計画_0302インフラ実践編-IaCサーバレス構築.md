# IaCサーバレス構築<br>( Lambda、API Gateway、IAM Role、CloudWatch Logs )

## 1. サーバレス構成とは？

サーバレス構成とは、サーバー台数やOS管理を意識せず、  
**必要な処理だけを実行するクラウド構成**です。

AWSでは主に以下を組み合わせます。

- **Lambda**：プログラム実行
- **API Gateway**：HTTP API公開
- **IAM Role**：権限付与
- **CloudWatch Logs**：ログ確認

```plaintext
利用者
  ↓ HTTPアクセス
API Gateway
  ↓
Lambda
  ↓
CloudWatch Logs
```

👉 インフラ管理負荷を下げ、開発スピードを上げやすい構成です。

## 2. IaCで構築する意義

管理画面で手動作成も可能ですが、実務ではIaC化が重要です。

* 同じ環境を再現できる
* 開発 / 本番環境を統一しやすい
* レビューできる
* 変更履歴をGitで管理できる
* 手作業ミスを減らせる

```plaintext
手動構築
担当者依存・再現しづらい

IaC構築
コード化・誰でも同じ環境
```

## 3. Lambda を IaC で定義する

Lambda はコードを実行するサービスです。  
Lambda は実行時に CloudWatch Logs の LogGroup を自動作成しますが、保持期間はデフォルトで無期限のため、IaC で明示的に設定するのが実務では一般的です。


### 例：最小構成

```yaml
Resources:
  SampleFunction:
    Type: AWS::Lambda::Function
    Properties:
      FunctionName: sample-function
      Runtime: python3.12
      Handler: index.lambda_handler
      Role: arn:aws:iam::123456789012:role/sample-role
      Code:
        ZipFile: |
          def lambda_handler(event, context):
              return {"statusCode": 200}
```

### 主な項目

| 項目           | 内容     |
| ------------ | ------ |
| FunctionName | 関数名    |
| Runtime      | 実行言語   |
| Handler      | 開始関数   |
| Role         | 実行権限   |
| Code         | ソースコード |

## 4. API Gateway を IaC で定義する

Lambda を外部公開する際に利用します。

```plaintext
ブラウザ
 ↓
API Gateway
 ↓
Lambda
```

### 利用例

* Web API
* 社内ツールAPI
* LINE Bot連携
* 外部システムWebhook

### 定義イメージ

```yaml
Resources:
  SampleApi:
    Type: AWS::ApiGateway::RestApi
    Properties:
      Name: sample-api
```

👉 API Gateway が入口、Lambda が処理担当です。

API Gateway は RestApi を作成しただけでは動作しません。  
実際には以下のリソースも必要です。

- Resource（パス）：例 /hello
- Method（GET/POST など）
- Integration（Lambda との紐付け）
- Lambda 側の Invoke 権限（AWS::Lambda::Permission）
これらを組み合わせて初めて API が動作します。 

## 5. IAM Role を IaC で定義する

Lambda が他サービスへアクセスするには権限が必要です。

例：

* CloudWatch Logsへ出力
* S3へ保存
* DynamoDB参照

```yaml
Resources:
  LambdaRole:
    Type: AWS::IAM::Role
    Properties:
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Service: lambda.amazonaws.com
            Action: sts:AssumeRole
```

```plaintext
Lambda
 ↓ 権限
IAM Role
 ↓
S3 / Logs / DB
```

👉 AccessKey直書きではなく、Role利用が基本です。

## 6. CloudWatch Logs を確認する

Lambda実行結果やエラー内容は Logs で確認します。

例：

* print出力
* エラー原因
* 実行回数
* 実行時間

```plaintext
Lambda実行
 ↓
CloudWatch Logs
```

IaCで明示的にログ保持期間設定も可能です。

```yaml
Resources:
  LogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: /aws/lambda/sample-function
      RetentionInDays: 14
```

## 7. Parameter で環境差分を切替える

開発環境と本番環境で設定を変える場合に使います。

```yaml
Parameters:
  EnvName:
    Type: String
```

利用例：

* 関数名
* メモリサイズ
* タイムアウト秒数

```plaintext
dev → 小さく安価
prod → 高性能
```
環境差分は Parameter だけでなく、Mappings や Conditions を組み合わせるとより柔軟に dev/prod を切り替えられます。

## 8. 実行コマンド

### 構文確認

```bash
aws cloudformation validate-template \
--template-body file://template.yaml
```

### デプロイ

```bash
aws cloudformation deploy \
--template-file template.yaml \
--stack-name serverless-stack \
--capabilities CAPABILITY_NAMED_IAM \
--parameter-overrides EnvName=dev
```

### 削除

```bash
aws cloudformation delete-stack \
--stack-name serverless-stack
```

## 9. 実務でよくある構成

```plaintext
API利用者
   ↓
API Gateway
   ↓
Lambda
   ├─ S3保存
   ├─ DynamoDB保存
   └─ CloudWatch Logs
```
API Gateway が Lambda を呼び出すには、Lambda 側に`AWS::Lambda::Permission` を設定する必要があります。


## 10. よくある失敗例

| 失敗         | 原因              |
| ---------- | --------------- |
| Lambda実行失敗 | IAM権限不足         |
| Lambdaタイムアウト | デフォルト3秒のまま |
| VPC内Lambdaが遅い | ENI作成待ち |
| APIアクセス不可  | API Gateway設定漏れ |
| 外部APIに接続不可 | SGの egress 制限 |
| ログが見れない    | Logs権限不足        |
| dev/prod混在 | 環境分離不足          |

## 11. まとめ

* Lambda は処理実行担当
* API Gateway は公開入口
* IAM Role は権限管理
* CloudWatch Logs は調査用
* IaC化で再現性・安全性が高まる

👉 サーバレス構成は **小さく始めて拡張しやすい実務向け構成** です。

## 12. 重要用語説明

| 用語 | 説明 |
|---|---|
| Lambda( 関数実行サービス )<br>AWS Lambda | サーバー管理不要でプログラムを実行できるサービス。 |
| API Gateway( API管理 )<br>Amazon API Gateway | API公開、認証、制御を行うマネージドサービス。 |
| CloudWatch( 監視サービス )<br>Amazon CloudWatch | メトリクス監視、ログ確認、アラーム通知を行うサービス。 |
| IAM Role( 権限ロール )<br>Identity and Access Management Role | AWSサービスへ権限を委任する仕組み。 |
| CloudWatch Logs( ログ保存機能 )<br>CloudWatch Logs | CloudWatchでログを保存・検索する機能。 |
| Runtime( 実行環境 )<br>Runtime | Lambdaで使用する言語実行環境。 |
| Handler( 開始関数 )<br>Handler | 実行開始地点となる関数。 |
| Parameter( 入力値 )<br>Parameter | 外部から渡される設定値や入力値。 |
| Deploy( 反映 )<br>Deploy | コードや設定をAWS環境へ適用すること。 |
| Serverless( サーバーレス )<br>Serverless | サーバー管理を意識せず利用できる実行形態。 |
| Log( 動作記録 )<br>Log | システムの動作記録やエラー情報。 |
| Trigger( 起動条件 )<br>Trigger | Lambdaなどを起動するきっかけとなるイベント。 |
| Timeout( 制限時間 )<br>Timeout | 処理実行の上限時間。 |
| Endpoint( 接続先URL )<br>Endpoint | APIなどの利用先URL。 |
