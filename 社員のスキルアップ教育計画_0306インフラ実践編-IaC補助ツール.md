# IaC補助ツール<br>( Amazon Q、テンプレート生成、エラー解析、IAM Policy支援 )

## 1. IaC補助ツールとは？

IaC は便利ですが、最初は以下でつまずきやすいです。

- YAML の書き方が分からない
- CloudFormation エラーが読めない
- IAM Policy が難しい
- AWS構成が思いつかない
- 設定ミスが不安

そこで役立つのが **AI補助ツール** です。  
AWS公式の生成AI支援サービスが **Amazon Q** です。

👉 人が設計し、AIが補助する形が実務的です。

## 2. Amazon Qとは？

Amazon Q は AWS公式のAIアシスタントです。  
AWSサービスの質問、コード生成、設計補助などに使えます。

### 主な用途

- CloudFormation テンプレート作成支援
- Terraform サンプル生成
- IAM Policy たたき台作成
- エラー原因調査
- AWS構成相談
- ドキュメント要約

```plaintext
利用者質問
   ↓
Amazon Q
   ↓
回答・コード提案
```

## 3. IaC生成支援

例えば以下のように依頼できます。

```plaintext
S3バケットを作る CloudFormation YAML を作成してください。
Tag に Project と Env を付けてください。
```

### 得られる効果

* 雛形作成が速い
* 書式ミスを減らせる
* 初学者でも開始しやすい

### ただし注意

AI生成コードはそのまま本番投入せず、
必ずレビューします。

## 4. テンプレート補助

既存テンプレート改善にも使えます。

### 例

```plaintext
この YAML を可読性高く整理してください
Parameters 化してください
prod / dev 切替できるようにしてください
```

### よくある改善点

* Parameters追加
* Outputs追加
* コメント整備
* リソース分割
* 命名統一

## 5. エラー解析支援

CloudFormation は失敗時に英語エラーが出ることがあります。

### 例

```plaintext
Resource handler returned message:
AccessDenied
```

これを Amazon Q に貼って、

```plaintext
この CloudFormation エラー原因と対処を教えてください
```

と依頼できます。

### 効果

* 原因特定が速い
* IAM不足か判別しやすい
* 修正候補が分かる

## 6. IAM Policy作成支援

IAM は初心者最大の壁のひとつです。

### 例

```plaintext
Lambda が S3 の特定バケットだけ読める
IAM Policy を作ってください
```

### 出力例イメージ

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::sample-bucket/*"
    }
  ]
}
```

### 注意

* 権限過剰になっていないか確認
* Resource範囲確認
* 本番前レビュー必須

## 7. 実務で強い使い方

## 7-1 雛形生成 → 人が仕上げる

```plaintext
Amazon Qで下書き
 ↓
人が設計確認
 ↓
修正
 ↓
デプロイ
```

## 7-2 エラー時の一次調査

```plaintext
失敗ログ確認
 ↓
Amazon Qに質問
 ↓
原因候補整理
 ↓
実修正
```

## 7-3 学習用途

```plaintext
Lambda + SQS のCloudFormation例を見せて
```

などで学習速度が上がります。

## 8. 実行コマンド( 最終確認は人が行う )

### 構文確認

```bash
aws cloudformation validate-template \
--template-body file://template.yaml
```

### デプロイ

```bash
aws cloudformation deploy \
--template-file template.yaml \
--stack-name ai-generated-stack \
--capabilities CAPABILITY_NAMED_IAM
```

### 状態確認

```bash
aws cloudformation describe-stacks \
--stack-name ai-generated-stack
```

## 9. AI活用の注意点

| 注意点      | 内容             |
| -------- | -------------- |
| 機密情報入力禁止 | 社内情報や秘密鍵は入力しない |
| 鵜呑み禁止    | 誤回答の可能性あり      |
| 古い情報注意   | 最新仕様確認する       |
| 権限過剰注意   | IAMは特に要確認      |
| 本番直投入禁止  | 必ず検証する         |

## 10. Amazon Qが向く作業 / 向かない作業

| 向く       | 向かない     |
| -------- | -------- |
| 雛形作成     | 最終承認     |
| 学習質問     | 本番判断     |
| エラー整理    | 責任判断     |
| Policy草案 | セキュリティ確定 |
| リファクタリング | 完全自動設計   |

## 11. まとめ

* Amazon Q は IaC学習と作業速度向上に強い
* YAML雛形生成に便利
* エラー解析支援に有効
* IAM Policy 草案にも使える
* 最終判断は人間が行う

👉 **AIは代行者ではなく、優秀な補助者として使うのが正解です。**

## 12. 重要用語説明

| 用語 | 説明 |
|---|---|
| Amazon Q( AWS公式AI支援 )<br>Amazon Q | AWS利用や開発を支援するAIアシスタント。 |
| IaC( インフラコード管理 )<br>Infrastructure as Code | インフラ構成をコードで管理する考え方。 |
| CloudFormation( AWS構成管理 )<br>AWS CloudFormation | AWSリソースをコードで管理するIaCサービス。 |
| Terraform( 汎用IaCツール )<br>Terraform | 複数クラウドに対応したIaCツール。 |
| IAM Policy( 権限制御ルール )<br>Identity and Access Management Policy | AWS操作権限をJSON形式で定義するルール。 |
| Prompt( 指示文 )<br>Prompt | AIへ与える質問や命令文。 |
| Review( レビュー )<br>Review | 人が内容を確認し品質を担保する工程。 |
| Validation( 検証 )<br>Validation | 構文や内容の正しさ確認。 |
| YAML( 設定記述形式 )<br>YAML Ain't Markup Language | 人が読みやすい設定ファイル形式。 |
