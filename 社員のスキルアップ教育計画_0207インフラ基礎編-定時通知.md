## 定時処理と通知の自動化<br>～EventBridge × Lambda × SNS で業務連絡を自動化する～

企業のシステム運用では、毎日決まった時間に行う作業や、特定のイベントをきっかけに実行したい処理が数多く存在します。  
- 毎朝のシステム状態チェック  
- 日報や業務連絡の自動送信  
- バッチ処理の定時実行  
- エラー発生時の担当者への通知  

これらを手作業で行うと、担当者の負担が増えたり、ミスが発生したりします。  
そこで役立つのが **EventBridge（スケジュール）× Lambda（処理）× SNS（通知）** の組み合わせです。  

この章では、実際の業務シーンを交えながら、AWSの代表的な定時処理サービス **EventBridge**( イベント連携 )と、通知・メッセージ配送を担う **SNS**( Simple Notification Service )を、仕組みの流れがつかみやすいように整理して解説します。  

これらのサービスを使いこなすことで、複雑なタスクの自動化やスタッフへの迅速な情報伝達が可能になり、運用負荷の軽減とレスポンスの高速化が実現します。

---
### 1. AWS EventBridge とは？ <br>～決まった時間やイベントをきっかけに自動で動く仕組み～

- **概要**  
**EventBridge (イベントブリッジ)** は、AWS内外のイベントを受け取って、その情報を元に設定した処理( ターゲット )を自動実行するイベント連携サービスです。  

- **主な特徴**
  - AWSサービスタイプのイベント( EC2の起動やS3のファイル追加など )をトリガーにできる
  - Cron 式で定時スケジュールを設定可能
  - 複数のサービスをつなげて自動化が容易
  - サーバーレスで運用負荷が少ない  

  EventBridge は「イベントバス」という受け皿にイベントが流れ込み、ルールが条件に合うイベントを選び、ターゲットへ転送する仕組みになっています。

- **利用シーンの例**
  - 毎朝のデータバックアップ処理を自動実行する  
  - 新しいファイルがアップロードされたら処理を開始  
  - システムログ異常時にアラート通知  

```plaintext
[イベント発生源]-EC2起動、定期時間など
      |
      ▼
[EventBridge(ルール)]-条件に合うイベントを抽出・転送
      |
      ▼
[処理(Lambda/SQS/SNSなど)]
```

---
### 2. AWS Lambda とは？<br>～必要なときだけ動く処理エンジン～

- **概要**  
Lambda は、サーバーを用意せずにコードを実行できるサービスです。  
EventBridge から呼び出され、実際の処理を担当します。  

- **主な特徴**  
  - サーバー管理が不要
  - 必要なときだけ実行されるためコスト効率が高い
  - Python / Node.js など複数言語に対応
  - サービスとの連携が容易
  - 最大実行時間は 15 分で、長時間処理には向かない

- **利用シーンの例**  
  - 日報の自動生成（データ集計 → テキスト作成）
  - S3 にアップロードされたファイルの加工処理
  - API 呼び出しや外部サービス連携
  - エラー検知時のログ解析

```plaintext
[EventBridge / S3 / API] 
          |
          ▼
  [Lambda（コード実行）]
          |
          ▼
   [SNS / S3 / DB]
```

---
### 3. AWS SNS とは？<br>～作成したメッセージを複数の宛先に届ける通知基盤～

- **概要**  
**SNS（Simple Notification Service）** は、メッセージを複数の購読者に一斉配信するサービスです。

- **主な特徴**  
  - メール、SMS、HTTP、Slack（Webhook）など多様な配信先
  - Topic 単位で購読者を管理
  - 高いスケーラビリティと信頼性
  - 他サービスと連携してリアルタイム通知が可能
  - メッセージサイズは最大 256 KB（大きなデータ送信には不向き）

- **利用シーンの例**  
  - システム障害発生時の管理者へのメール通知
  - 日報や業務連絡の一斉配信
  - Slack へのアラート通知
  - バッチ処理完了の通知

```plaintext
[発信元（Lambda / EventBridge）]
            |
            ▼
        [SNS Topic]
  ┌────────────────────────┐
  │ メール｜SMS｜Slack｜HTTP │
  └────────────────────────┘
            |
            ▼
     [複数購読者へ同時配信]

```

---
### 4. EventBridge × Lambda × SNS の連携例<br>～定時処理から通知までを一連の流れで自動化する～

- **概要**  
EventBridge、Lambda、SNS を組み合わせることで、「決まった時間に処理を実行し、その結果を通知する」という一連の業務フローを自動化できます。  
この構成は、日報送信・バッチ処理完了通知・エラーアラートなど、多くの運用シーンで利用される代表的なパターンです。

- **主な特徴**
  - **スケジュール実行 → 処理 → 通知** の流れを一つの仕組みで完結できる  
  - サーバーレス構成のため、運用負荷が少なくコストも低い  
  - 各サービスが疎結合のため、処理内容や通知先を柔軟に変更できる  
  - 小規模な業務連絡から大規模システムの定時処理まで幅広く対応

- **利用シーンの例**  
**例：毎朝 8:00 に日報を自動送信する仕組み**
1. EventBridge（スケジュール）  
毎朝 8:00 に Lambda を起動するよう Cron 式で設定します。  
2. Lambda（処理）  
昨日のデータを取得し、日報テキストを作成します。
3. SNS（通知）  
作成した日報をメールや Slack に配信します。

この流れにより、担当者が毎朝手作業で行っていた日報作成・送信を完全自動化できます。

```plaintext
[EventBridge（毎朝 8:00に実行）]
          │
          ▼
[Lambda（日報作成処理）]
          │
          ▼
[SNS（メール/Slackに配信）]
```

### 5. 実務でのポイント

- **小さく作って段階的に拡張する**  
  いきなり複雑な処理を作ると、原因調査や修正が難しくなります。まずは「固定メッセージを送る」など最小構成で動かし、徐々に機能を追加する方が安定します。

- **SNS の購読者管理を定期的に見直す**  
  異動・退職などで不要な宛先が残ると、情報漏洩や誤通知につながります。定期的に購読者リストを確認し、必要なメンバーだけが受け取る状態を維持することが重要です。

- **コストはほぼ無料だが、通知頻度には注意する**  
  EventBridge や Lambda は無料枠で収まることが多く、メール通知も無料です。ただし、通知が大量に発生すると SNS のコストが増える場合があります。必要な通知だけを送る設計が大切です。

- **IAM 権限は最小限にする**  
  EventBridge → Lambda、Lambda → SNS のように、必要な権限だけを付与します。広すぎる権限は誤操作や情報漏洩のリスクを高めるため、最小権限の原則を徹底します。

---
### 6. まとめ

- **EventBridge：**タイミングを決める（いつ動くか）  
- **Lambda：**実処理を実行する（何をするか）  
- **SNS：**結果を届ける（誰に知らせるか）  

この 3 つを組み合わせることで、日報送信や定時連絡などの業務を自動化し、運用負荷を大きく減らすことができます。

---
### 7. 重要用語説明

| キーワード | 説明 |
|------------|-------|
| EventBridge (イベントブリッジ) | AWSのイベント連携サービス。イベント発生をトリガーに自動処理を実行 |
| Lambda (ラムダ) | サーバー不要でコードを実行できるサービス |
| SNS<br>Simple Notification Service | AWSのメッセージ配信サービス。メール・SMS等へ通知可能 |
| SMS<br>Short Message Service | 携帯電話番号宛にテキストメッセージを送信する仕組み。SNSでは通知手段の一つとして利用可能 |
| Rule( ルール ) | EventBridgeでイベントを検出し、動作を指定する設定 |
| Target( ターゲット ) | イベントに応じて処理されるAWSサービスやエンドポイント |
| Topic( トピック ) | SNSでメッセージ配信先をグループ管理する単位 |
| Cron式 | スケジュール設定で用いる日時指定フォーマット |
| IAM( 管理コンソール )<br>Identity and Access Management | AWSリソースの権限管理サービス |
