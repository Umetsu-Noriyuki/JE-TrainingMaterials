## ストレージと高速配信の基礎
S3とCloudFrontで作る効率的なデータ管理と配信（AWS）  
  
AWSのストレージ（Storage）と配信（Content Delivery）基盤を理解することは、システムの効率的な運用とユーザー体験の向上に直結します。  
ここでは、**S3（Simple Storage Service）**という高耐久なオブジェクトストレージと、  
コンテンツを高速に配信するための**CloudFront（コンテンツデリバリーネットワーク - CDN）**の基本を学び、  
実務での利用イメージをつかんでいきましょう。

---

## 1. S3（Simple Storage Service）とは？ クラウド上の「ファイル置き場」

- **何をする？**  
  大量のファイルやデータをインターネット経由で保存・取得できるサービスです。  
  例：画像、動画、ログ、バックアップデータの保管に最適。

- **特徴**  
  - 非常に高い耐久性（99.999999999%）  
  - 保存容量は無制限  
  - アクセス制御（IAM連携）で安全管理  
  - 階層構造ではなく、「バケット」と呼ばれる箱の中に、キー（名前）付きのオブジェクトを格納  
  **補足：** 「photos/ocean.jpg」のように「/」を含めて一つの名前として扱うため、大量のデータの中からでも名前さえ分かれば即座に取り出せるのが強みです。  
  (※操作画面ではフォルダのように見えますが、内部的にはすべてのデータが並列に管理されるフラットな構造です）
  - 利用頻度に合わせて「保存料金」を安くできる  
  （例：滅多に見ないバックアップは「S3 Glacier」という安い場所に保管する）

- **実務での使い方**  
  - Webサイトの画像や動画保存  
  - バックアップデータの保存・復元  
  - システムログの蓄積  

- **簡単なイメージ**  

```plaintext
【ユーザーから見たイメージ】
（フォルダ分けされているように見えます）
S3 バケット（例: my-company-assets）
    ├── logo.png
    ├── backup/
    │     ├── backup-2024-06-01.zip
    │     └── backup-2024-06-08.zip
    └── logs/
          ├── app-log-2024-06-05.txt
          └── app-log-2024-06-06.txt

【内部的なデータの持ち方】
システムの中では、以下のように「パスを含めた一つの名前」として
シンプルに管理されています。
・ logo.png
・ backup/backup-2024-06-01.zip
・ logs/app-log-2024-06-05.txt
```

## 2. CloudFrontとは？ コンテンツを高速に世界中に届ける仕組み（CDN）

- **何をする？**  
  世界中に設置されたサーバー（エッジロケーション）を使い、静的・動的コンテンツを利用者の近くから高速配信します。

- **特徴**  
  - AWSや自社サーバーのコンテンツをキャッシュし高速配信  
  - 低遅延でユーザー体験を向上  
  - SSL/TLS対応で通信を安全に  
  - 不正アクセス防止のセキュリティ機能  

- **実務の利用シーン**  
  - Webサイトの画像や動画を高速配信  
  - **Webサーバー（ALB/EC2）の前段に置いて負荷を軽減**  
  ※S3だけでなく、動的なWebサーバーからのレスポンスも高速化できます
  - ソフトウェアのダウンロード配信  
  - APIレスポンスのキャッシュを通じた負荷分散  

- **簡単な図解イメージ**  

```plaintext
【ユーザーA (日本)】                         【ユーザーB (アメリカ)】
      │ (近い！速い)                                 │ (近い！速い)
      ▼                                            ▼
[CloudFront エッジロケーション(東京)]   [CloudFront エッジロケーション(米国)]
      │                                            │
      └──────────────────────┬─────────────────────┘
                             │ (初回だけ取りに行く)
                             ▼
                      [オリジンサーバー]
                      (例：東京にあるS3)
```

- **リージョンとエッジロケーションの構造的違い**
  - **物理的な配置の独立性:低遅延**   
AWSリージョンは、世界でも限られた特定の広域エリア（東京、バージニアなど）に集約して設置されています。一方で、エッジロケーションはリージョンの所在地に縛られず、それ以外の主要都市（大阪、ソウル、ロンドン、ニューヨークなど）に独立して設置されています。  
ユーザーは「海を越えた遠くのリージョン」ではなく、「自分の住む街の近くにあるエッジ」と通信するため、物理的な距離が劇的に短縮され、レスポンスが高速化します。

  - **多拠点によるトラフィック分散:負荷軽減**  
  拠点の数は「数十箇所のリージョン」に対し、「数百箇所の無数のエッジ」という圧倒的な差があります。世界中からの膨大なアクセスを、各地域に分散配置されたエッジがリージョンの手前で食い止める構造になっているため、オリジンサーバー（S3など）までリクエストが到達する回数が激減し、負荷が抑えられます。

## 3. S3とCloudFrontを組み合わせるメリット

| 項目               | S3単体利用              | S3 + CloudFront 利用                      |
|--------------------|-----------------------|-----------------------------------------|
| アクセス速度       | オリジン（S3）から直接  | エッジロケーションから高速配信          |
| 負荷分散           | なし                    | 高トラフィックでも耐えられる            |
| コスト             | データ転送料高くなる場合あり | キャッシュ効果でコスト削減可能           |
| セキュリティ強化   | バケットを公開設定にする必要があり、リスクが残る | **S3を非公開にしたまま配信可能（OAC）**<br>「CloudFrontからのアクセスのみ許可」という鍵をかけられる   |
| ユーザー体験       | 遅延が発生する可能性あり  | 遅延が少なく快適な操作性を実現          |

**実務のポイント：**  
現在は **OAC（Origin Access Control）** という機能を使うのが主流です。これにより、S3バケットを「誰でも見られる状態」にせず、CloudFront経由の安全なアクセスだけを許可する設定が簡単にできます。

---

## 4. 実務で役立つポイント

- S3に公開用バケットを作成し、CloudFrontを経由して安全かつ高速に配信できます。  
- 静的Webサイトホスティングのベースとしても利用可能。  
- **ファイルを更新した際は「キャッシュの削除（Invalidation）」が必要**  
3のファイルを差し替えても、CloudFront側に古いデータがキャッシュとして残っていると、ユーザーには更新内容が反映されません。そのため、必要に応じて強制的にキャッシュをクリアする「無効化（Invalidation）」操作を行います。
- アクセス権限は必ずIAMとバケットポリシー（OACなど）で限定し、不要な公開は避けること。  
- 大量アクセスが予想される場合は必ずCloudFrontの導入を検討し、負荷とコスト対策を。

## 5. まとめと次ステップ

- **S3**はAWSの「安全で拡張性の高いファイル置き場」  
- **CloudFront**は世界中から高速に「コンテンツを届ける仕組み」  
- この2つは多くのAWS構成で必須の基礎サービスです。

## 6. 重要用語説明

| キーワード | 説明 |
|---|---|
| S3<br>Simple Storage Service | AWSのオブジェクトストレージサービス。大量のファイルをネット経由で保存可能。 |
| Bucket( バケット ) | S3内でのデータ保存単位。フォルダのようにデータをまとめる箱。 |
| Object( オブジェクト ) | S3に保存するファイルの単位。データ本体とメタ情報で構成。 |
| CloudFront | AWSのCDNサービス。世界中に分散したキャッシュサーバーで高速配信を実現。 |
| CDN<br>Content Delivery Network | コンテンツ配信ネットワーク。地理的に遠い利用者にも高速にデータを届ける仕組み。 |
| Edge Location( エッジロケーション ) | CDNが設置されている世界各地のサーバー。ユーザーに近いほど高速になる。 |
| IAM<br>Identity and Access Management | AWSの認証・権限管理サービス。安全なアクセス制御に必須。 |
| Origin（オリジン）| **配信元のデータがある場所**（S3バケットやWebサーバーなど）のこと。|
| OAC<br>Origin Access Control | **オリジンを守るためのアクセス制御。** 特定のCloudFront経由以外のアクセスを拒否し、S3などを安全に保護する仕組み。|
| Cache（キャッシュ） | 一度読み込んだデータをエッジロケーションに一時保存し、2回目以降のアクセスを速くする仕組み。|
