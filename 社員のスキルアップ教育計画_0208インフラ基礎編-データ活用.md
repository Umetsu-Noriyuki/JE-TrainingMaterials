# データを集めて活用する仕組み<br>( AWSで学ぶデータ収集・保存・分析の流れ )
現代のITシステムでは、Webアクセス、IoT機器、業務データなど、  
日々大量のデータやイベントが発生しています。  

それらをそのまま放置するのではなく、効率良く処理・連携し、  
分析につなげることで、売上向上・業務改善・障害検知などの  
ビジネス価値へ結びつけることが重要です。  

AWS には、この一連の流れを支える多くのサービスがあります。  

本章では、まず全体像を理解し、  
その後に各サービスの役割や実務での使われ方を学びます。
- 非同期イベント処理
- データストリーム配信
- ETL処理
- 大規模分析基盤へのデータ投入

## 1. まず全体像( AWSでのデータ活用フロー )
企業システムでは、データ活用は次の流れで行われることが一般的です。  
- 集める  
- 保存する  
- 整える  
- 分析する  

AWS では、この流れを以下のようなサービスで実現できます。  
また、重要データでは SQS を利用し、失敗時の再実行性を高める構成もよく使われます。  
```plaintext
リアルタイムデータ( Webログ・注文情報・IoTデータなど )
　　　↓
　　Firehose( 収集 )
　　　↓
　　S3( 保存 )
　　　↓
　　SQS( 保証 )
　　　↓
　　Glue( 変換 )
　　　↓
　　Redshift( 分析 ) ⇒ BIツール・SQLで可視化・意思決定
　　　　　　　　　　　　　　- Webアクセスログの分析
　　　　　　　　　　　　　　- 売上データの集計
　　　　　　　　　　　　　　- 顧客行動の可視化
　　　　　　　　　　　　　　- IoT機器の監視
　　　　　　　　　　　　　　- レポート作成の自動化
　　　　　　　　　　　　　　- 大量処理の分散実行
```

### サービスと役割一覧
| 工程 | サービス | 処理概要 |
|---|---|---|
| 収集 | Firehose | 大量データを継続的に取り込む<br>( イベント単位からある程度の処理単位に変換 ) |
| 保存 | S3 | 元データとして長期保存 |
| 保証 | SQS | 非同期処理としてキューイング<br>( 失敗した場合 再度実行も可能に ) |
| 変換 | Glue | 分析しやすい形へETL変換する       |
| 分析 | Redshift | SQL分析・BI連携 |

## 2. Firehose - データ収集担当
Firehose は、大量のログやイベントデータを受け取り、  
S3 などへ自動配送するストリーミング収集サービスです。  

配信時に、イベント単位のデータを一定量・一定時間ごとにまとめて保存できます。  
高頻度で発生するデータでも、後続処理しやすい単位で扱えます。  
名前の通り、消火栓のように、大量のデータを継続的に受け止め、送り出す役割として活用されます。

### 実務で役立つポイント
* サーバ管理不要で導入しやすい
* リアルタイム収集基盤に向き、大量データを継続的に取り込める
* 以前は Kinesis Data Firehose と呼ばれていました

## 3. S3( Simple Storage Service ) - データの長期保存担当
※S3の詳細は他章で扱っています。  
ここでは分析基盤で重要なイベント連携機能を補足します。

### S3イベント通知によるGlue連携
S3には、データのアップロードをきっかけに、他サービスを自動起動する **イベント通知機能** があります。  
この機能を利用し、データ保存後にGlueのETLジョブを自動実行する構成がよく利用されます。  

S3は単なる保存先にとどまらず、分析基盤の自動化と品質管理の要となるサービスです。

## 4. SQS( Simple Queue Service ) - システム連携・再実行担当
SQS は、処理依頼を一時的に預かるメッセージキューサービスです。  
後続システムが停止していても受付でき、失敗時の再処理にも向いています。

### 実務で役立つポイント
* アクセス集中を吸収でき、システムを疎結合にできる
* DLQ( デッドレターキュー )で失敗データは退避され、後に再実行もしやすいため、重要データの保護に向く

## 5. Glue - データ変換・ETL担当
Glue は、保存されたデータを分析しやすい形へ整える、サーバーレス ETL サービスです。  

**ETL とは以下の意味です。**
* Extract( 抽出 )
* Transform( 変換 )
* Load( 投入 )

### 実務で役立つポイント
* Lambdaより長時間処理に向き、変換処理に強い
* 定期バッチの自動化に向く

## 6. Redshift - 分析担当
Redshift は、大量データを高速に集計・分析できる、クラウド型データウェアハウスです。  

### 実務で役立つポイント
* 列指向DBのため集計処理に強く、SQLで大量データ分析が可能  
  そのため、レポート基盤として使いやすく、定期分析や経営指標集計に向く  
* 更新処理が遅いため、頻繁なデータ更新に弱い  
  そのため、本構成では Firehose でデータを一定単位にまとめ、Redshiftへ効率的に取り込んでいます。

## 7. 重要用語説明

| キーワード | 説明 |
| --- | --- |
| Firehose<br>Kinesis Data Firehose | 大量のストリーミングデータをリアルタイムに配信するサービス |
| S3<br>Simple Storage Service | AWSのオブジェクトストレージ。長期保管やデータ連携の拠点 |
| SQS<br>Simple Queue Service | AWSのメッセージキューサービス。非同期処理を支える |
| Glue | サーバーレスETLサービス。データの抽出・変換・読み込みを実行 |
| Redshift | クラウド型データウェアハウス。高速に大規模データを分析 |
| ETL<br>Extract Transform Load | データの抽出・変換・投入 |
