# IaC非同期データ構築<br>( SQS、S3、Firehose、Glue、IAM Role )

## 1. 非同期・データ連携構成とは？

非同期・データ連携構成とは、  
**処理をすぐ返す部分と、後で順番に処理する部分を分ける仕組み**です。

また、業務データやログデータを蓄積し、後から加工・分析しやすくします。

例：

- 注文受付後、裏側でメール送信
- アップロード後、画像変換
- アクセスログを保存・集計
- CSVを加工して分析基盤へ連携

```plaintext
利用者操作
   ↓
受付完了( 即時応答 )
   ↓
裏側で順次処理
```

👉 応答速度向上・障害分離・大量データ処理に強い構成です。

## 2. IaCで構築する意義

非同期構成は連携サービスが増えるため、
手動設定では全体像が見えにくくなります。

IaC化すると、

* キュー・保存先・権限を一括管理できる
* 同じ構成を再現できる
* 環境差分を管理できる
* レビューしやすい
* 障害時の構成確認がしやすい

```plaintext id="p9s0we"
手動構築
設定箇所が分散

IaC構築
全体構成をコード管理
```

## 3. SQS( 非同期キュー )

SQS はメッセージを一時保存し、
後続システムが順番に処理するサービスです。

### 利用例

* 注文データ処理
* メール送信待ち
* バッチ連携
* リトライ処理

```plaintext id="h7x4mf"
受付API
 ↓
SQS
 ↓
Worker(Lambda等)
```

### IaC定義例

```yaml id="n2w5jq"
Resources:
  SampleQueue:
    Type: AWS::SQS::Queue
```

👉 システム混雑時の緩衝材として有効です。

## 4. S3( データ保存 )

S3 はファイルやログなどを保存する代表的ストレージです。

### 利用例

* CSV保管
* 画像保管
* バックアップ
* アクセスログ保存

```plaintext id="r3m8zn"
各システム
 ↓
S3
```

### IaC定義例

```yaml id="b6v1tk"
Resources:
  DataBucket:
    Type: AWS::S3::Bucket
```

## 5. Firehose( データ配送 )

Firehose はストリームデータを自動配送するサービスです。

例：

* アプリログ → S3
* イベントデータ → Redshift
* 分析データ収集

```plaintext id="v8p3ua"
大量ログ
 ↓
Firehose
 ↓
S3 / 分析基盤
```

### 特徴

* 自動バッファ保存
* 大量データ向き
* 運用負荷が低い

## 6. Glue( ETL処理 )

Glue はデータ加工・変換サービスです。

ETLとは：

* Extract( 抽出 )
* Transform( 変換 )
* Load( 格納 )

### 利用例

* CSV整形
* 日付変換
* データ結合
* 分析用形式へ変換

```plaintext id="m0f7yr"
S3元データ
 ↓
Glue
 ↓
分析しやすい形へ変換
```

## 7. IAM Role( サービス連携権限 )

各サービスが連携するには Role が必要です。

例：

* Firehose → S3保存
* Glue → S3読取
* Lambda → SQS受信

```plaintext id="d5w2oq"
Glue
 ↓ 権限
IAM Role
 ↓
S3アクセス
```

AccessKey直書きではなく Role利用が基本です。

## 9. 実行コマンド

### 構文確認

```bash id="y8k2fh"
aws cloudformation validate-template \
--template-body file://template.yaml
```

### デプロイ

```bash id="w1n3dr"
aws cloudformation deploy \
--template-file template.yaml \
--stack-name async-stack \
--capabilities CAPABILITY_NAMED_IAM
```

### 削除

```bash id="c0m7ls"
aws cloudformation delete-stack \
--stack-name async-stack
```

## 10. よくある失敗例

| 失敗      | 原因       |
| ------- | -------- |
| メッセージ滞留 | Worker不足 |
| 保存失敗    | IAM権限不足  |
| データ欠損   | Glue変換ミス |
| 本番混在    | バケット分離不足 |

## 11. まとめ

* SQS は非同期処理の中心
* S3 は保存基盤
* Firehose は大量配送向け
* Glue は加工・変換担当
* IAM Role は安全な連携に必須
* IaC化で複雑な構成を安全管理できる

👉 非同期構成は **速い画面応答と強い裏側処理** を両立します。

## 12. 重要用語説明

| 用語 | 説明 |
|---|---|
| SQS( メッセージキュー )<br>Amazon Simple Queue Service | メッセージを一時保存し順番に処理するサービス。 |
| Firehose( データ配送 )<br>Amazon Data Firehose | ストリームデータを保存先へ自動配送するサービス。 |
| S3( オブジェクトストレージ )<br>Amazon Simple Storage Service | ファイルやログを保存するストレージサービス。 |
| Glue( ETLサービス )<br>AWS Glue | データ抽出・変換・格納を行うサービス。 |
| ETL( 抽出変換格納 )<br>Extract Transform Load | データを抽出し加工して保存する処理。 |
| IAM Role( 権限ロール )<br>Identity and Access Management Role | AWSサービスへ権限を委任する仕組み。 |
| Queue( 待機列 )<br>Queue | 処理待ちデータを順番に並べる仕組み。 |
| Deploy( 反映 )<br>Deploy | コードや設定をAWS環境へ適用すること。 |
| Worker( 処理担当 )<br>Worker | キューから取得して実処理を行う側。 |
| Batch( 一括処理 )<br>Batch Processing | まとめて実行する処理方式。 |
