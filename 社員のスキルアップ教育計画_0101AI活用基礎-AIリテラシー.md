# AIリテラシー
AIを「安全」かつ「賢く」使いこなすための基礎知識  

---

## 1. はじめに：AIは「魔法」ではなく「確率」

現在のAI( LLM )は、次に続く言葉を確率的に予測している巨大な計算機です。  
人間のような「思考」や「意識」ではなく、  
**膨大なデータに基づいた「もっともらしい推論」**を行っています。  

**「正しいこと」ではなく「それっぽいこと」を生成する存在**です。  

この前提を理解することが、AI活用の出発点になります。

---

## 2. AIはどのように振る舞うのか

AIは以下の特徴を持っています：

- 文脈から「最も自然な答え」を生成する  
- 正しさより「自然さ・一貫性」を優先する  
- 指示が曖昧だと、曖昧な回答になる  

**AIの出力は「入力( 問い方 )」に強く依存する**

---

## 3. AIの弱点

AIは万能ではありません。  

* **ハルシネーション( 幻覚 )**  
　→ 事実ではない内容をそれらしく生成する  
* **情報のカットオフ**  
　→ 最新情報に弱い場合がある  
* **論理のミス**  
　→ 複雑な思考で誤りが出る  

**重要な判断は必ず人間が行う**

---

## 4. セキュリティと倫理
AIは非常に便利なツールですが、社内利用では以下を厳守してください。

* 機密情報の入力禁止  
  - AIは入力内容を外部サーバーで処理する可能性があり、  
    入力した情報が学習やログに利用されるリスクがある
  - エンタープライズAI であれば、上記のリスクはなくなり、
    機密情報やソースコードなどを問いに貼り付けることが可能
* 著作権への配慮  
  - 生成内容が既存の著作物に類似する可能性があるため、そのまま利用しない  
* 最終責任は人間  
  - 誤情報をそのまま利用すると、業務事故や信用低下につながる  

**「便利さ」よりも「リスク管理」を優先する必要がある**

---

## 5. AIの精度は「問い方」で決まる
 
### ■ 基本原則

- 曖昧な質問 → 曖昧な回答  
- 具体的な質問 → 精度の高い回答  

AIは「意図」を推測して補完するため、  
曖昧な問いでは誤った前提で回答が生成される可能性があります。

**問いの質＝出力の質**

---

### ■ 例

❌ JavaScriptを教えて  

⭕ 未経験者向けに、具体例付きでJavaScriptを説明してください  

違いは「具体性」と「目的」

### ■ 結論

**業務では、具体性と目的を明確にした問いを基本とすること**
- 改善レベル：日常的な確認・軽い調査  
- 実務レベル：成果物を出す場合  
  「具体性」と「目的」を必ず明確にすること  
  ( 例：対象者・前提条件・期待する出力 )

---

## 6. プロンプトの基本

AIへの指示は、次の4つの要素で構成されます：
```
新人研修の講師として説明してください。  ⇒ Role( ロール=役割 )
対象：プログラミング未経験者            ⇒ Context( 背景 )
JavaScriptの基礎を教えてください。      ⇒ Task( 命令 )
・具体例あり                            ⇒ Constraint( 制約 )
・専門用語は最小限                      ⇒ Constraint( 制約 )
```
**すべて使う必要はない。足りなければ足す**

---

## 7. ロールの考え方

**ロールは必須ではない。基本は指定しなくてよい。**

AIは指示がない場合、無難で平均的な回答を生成します。

- 誰向けかわからない  
- 専門性がブレる  
- 表現が安定しない  

**より専門的な回答を得たい場合、ロールを指定します。**

---

### ■ ロールの役割

**回答の前提条件を固定する**

例：

- 教育担当 → わかりやすく説明する  
- エンジニア → 技術的に正確に説明する  
- コンサル → 論理的に整理する  

---

### ■ ロール有無の違い

| 観点 | ロールなし | ロールあり |
|------|------------|------------|
| 出力の安定性 | ややブレる | 安定する |
| 専門性 | 一般的 | 指定方向に寄る |
| 分かりやすさ | 状況依存 | コントロール可能 |
| 再現性 | 低い | 高い |
| 初心者の使いやすさ | 高い | やや低い |
| 柔軟性 | 高い | やや固定される |
| 指示コスト | 低い | 少し増える |
| AIの挙動 | デフォルト最適化 | 条件付き最適化 |
| 会話の一貫性 | 弱い | 強い |

---

### ■ 注意

ロールを指定しても、AI自体が学習するわけではありません。  
会話内での振る舞いが変わるだけです。

---

## 8. AI活用の成長ステップ
AIの活用レベルは「問いの内容」で決まり、問いを変えることで段階的に成長する。

### レベル1 NG ... 答えを聞く
> JavaScriptを教えて  

### レベル2 改善 ... 理解する
> 未経験者向けに、具体例付きでJavaScriptを説明してください  

### レベル3 実務レベル ... 思考を引き出す
> 新人研修用に、JavaScriptの基本を説明してください。  
> 比喩・注意点・実務での使い所を含めてください。  

**常にレベル3( 思考を引き出す問い )を意識すること**

---

## 9. AIの使い分け
 
**「調べる」はGemini、「考える」はChatGPTで使い分ける。**  

| 項目 | <a href="https://chat.openai.com/" target="_blank">ChatGPT</a> | <a href="https://gemini.google.com/" target="_blank">Gemini</a> |
|------|--------|--------|
| 特徴 | 思考・対話型 | 検索・情報型 |
| 強み | 深掘り・構造化 | 最新情報・収集 |
| 弱み | 最新性に弱い場合あり | 思考の深さはやや弱い |
| 向いている用途 | 学ぶ・考える | 調べる |
