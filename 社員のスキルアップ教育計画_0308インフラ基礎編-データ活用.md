# イベント処理・連携・分析<br>( AWSで実現する効率的なデータ活用基盤 )
ITシステムでは、多様なイベントやデータが日々発生します。  
これらを効率的に処理・連携し、分析・活用に繋げる基盤作りは重要な役割です。  

本章では、AWSの主要サービスである **SQS（メッセージキュー）**、**EventBridge（イベント連携）**、  
**Firehose（データストリーム配信）**、**Glue（ETL処理）**、**Redshift（データウェアハウス）**、  
そして **SNS（通知配信・Pub/Subメッセージング）** の概要と使いどころを解説します。  

---

## 1. SQS（Simple Queue Service）- 非同期メッセージ処理の基盤

**SQS** は分散システム間でのメッセージ送受信を安全かつ確実に行うサービスです。

- **役割**: メッセージをキュー（列）にため、順序制御・再送制御を行いながら非同期処理を実現
- **実務での利用例**:  
  - Webアプリの注文データを処理バッチへ連携  
  - バックエンドの各処理を独立して実行し、負荷分散する  
- **特徴**:  
  - フォールトトレラント（障害にも強い）  
  - 最大14日間メッセージ保持可能  
  - ロングポーリングで無駄なアクセスを減らせる

```plaintext
Webアプリ → SQS (メッセージキュー) → バックエンド処理
```

---

## 2. EventBridge - イベント駆動連携のハブ

**EventBridge** は AWSサービスやアプリケーションから発生するイベントを受け、  
条件に応じて処理先へルーティングして連携を自動化するサービスです。

- **役割**: イベント中心のアーキテクチャ実現、サービス連携の自動化  
- **実務での利用例**:  
  - AWSリソースの変更通知で処理開始  
  - SaaSや自社サービス間のイベント連携  
- **特徴**:  
  - ルールベースで柔軟に連携先を選定可能  
  - 複雑な処理もイベント単位で分割できる

```plaintext
EC2起動イベント → EventBridge → Lambda / SQS / SNS などへ連携
```

---

## 3. Firehose（Kinesis Data Firehose）- リアルタイムデータストリーム配信

**Firehose** は大量のストリーミングデータをAWSの保存先にリアルタイムで配信するサービスです。

- **役割**: データ収集～保存をリアルタイムかつ自動で実現  
- **実務での利用例**:  
  - Webログ・IoTデータの収集と保存（S3やRedshiftなど）  
  - データレイク更新の自動化  
- **特徴**:  
  - データ変換や圧縮も可能  
  - 高いスケーラビリティで大量データに対応

```plaintext
リアルタイムデータ → Firehose → S3 / Redshift / Elasticsearch
```

---

## 4. Glue - データ変換・結合のETL処理

**Glue** はETL（Extract, Transform, Load）処理を簡単にサーバーレスで実行できるデータ統合サービスです。

- **役割**: データ準備作業を効率化し、多種多様なデータを分析可能に  
- **実務での利用例**:  
  - 複数データソースの統合と整形  
  - データ品質チェックや更新処理  
- **特徴**:  
  - スケジュール実行やトリガー連携が可能  
  - コード生成機能で初心者でも扱いやすい

---

## 5. Redshift - 大規模データ分析のデータウェアハウス

**Redshift** は大量の構造化データを高速に分析できるクラウド型データウェアハウスです。

- **役割**: BIやデータ分析基盤の中心として大規模データを効率的に集約  
- **実務での利用例**:  
  - 売上データの月次・年次分析  
  - 複雑なSQLクエリを使った業務レポート作成  
- **特徴**:  
  - MPP(大規模並列処理)により高速分析  
  - S3やGlueとの連携でデータ取り込みが容易

```plaintext
データ分析基盤  
└─ Redshift ← AWS Glue ← S3
```

---

## 6. SNS（Simple Notification Service）- 通知とPub/Subメッセージング

**SNS** はシンプルにメッセージを多方向に配信するためのサービスです。

- **役割**: メッセージのプッシュ通知や複数サブスクライバーへの配信  
- **実務での利用例**:  
  - システム障害通知を管理者へメールやSMS送信  
  - イベントによるLambdaやSQSへのメッセージ送信  
- **特徴**:  
  - 複数プロトコル（メール、SMS、HTTPなど）に対応  
  - Pub/Subモデルでイベント駆動開発に最適

---

## 7. まとめ

- **SQS** は非同期で信頼性の高いメッセージキュー  
- **EventBridge** はイベント連携の自動化ハブ  
- **Firehose** はリアルタイムデータ配信と保存  
- **Glue** はデータ統合のETLサービス  
- **Redshift** は大規模データ分析のためのデータウェアハウス  
- **SNS** は通知やメッセージ配信の多目的サービス  

これらの連携によって、AWS上で高度なイベント処理とデータ分析基盤を効率よく構築できます。  
システム設計時には、処理の非同期分離やデータ活用の目的に応じて適切なサービスを選択してください。

---

## 8. 重要用語説明

| キーワード | 説明 |
|---|---|
| SQS<br>Simple Queue Service | AWSのメッセージキューサービスで非同期処理に利用 |
| EventBridge | AWSのイベントバスサービス。AWSやSaaSイベントを連携 |
| Firehose<br>Kinesis Data Firehose | 大量のストリーミングデータをリアルタイム配信するサービス |
| Glue | AWSのETLサービス。データ変換や統合をサーバーレスで実施 |
| Redshift | AWSのデータウェアハウス。大規模データの高速SQL分析可能 |
| SNS<br>Simple Notification Service | 複数チャネルへの通知・Pub/Subメッセージ配信サービス |
| ETL | Extract, Transform, Loadの略。データの抽出・変換・格納処理 |
| Pub/Sub | Publish/Subscribe の略。発行者と購読者のメッセージ配信モデル |
