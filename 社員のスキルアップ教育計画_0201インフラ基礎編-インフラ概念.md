# インフラ概念( Infrastructure )

## 1. インフラ( Infrastructure )とは何か？ - ITシステムを支える土台の基本構造

インフラ( Infrastructure )は、ITシステムの「土台」となるネットワークやサーバー、ストレージなどの設備を指します。  
この土台がなければ、アプリやサービスは動きません。  

近年は、このようなインフラを自社で保有するのではなく、クラウド上で提供されるサービスを利用する形が主流になっています。  
以降は代表例として、AWS( Amazon Web Services )を例に説明します。  

```plaintext
利用者
   ↓
Web画面 / スマホアプリ
( 利用者が直接使う部分 )
-------------------------
インフラ( 土台 )
   ├── EC2( サーバー )
   ├── VPC( ネットワーク )
   ├── S3 ( 保存領域 )
   └── RDS( データベース )
```

## 2. なぜ自前サーバーではなくAWSを使うのか？ - クラウドの利点と効率化

自社でサーバーを持つと、多くの手間やコスト、管理負担が発生します。  
AWSを利用すると、以下のようなメリットがあります。

- 初期費用を抑え、必要な分だけ使える  
- 運用・保守の負担が減る  
- 世界中の拠点から安定したサービス利用が可能  
- スケールアップ・スケールダウンが自由自在  

## 3. AWSの主要3大サービス - システムの土台を理解するための基礎知識

AWSには200以上のサービスがありますが、まずは以下の3つを押さえましょう。

| サービス名 | 説明 | 役割イメージ |
|------|------|------|
| EC2<br>Elastic Compute Cloud | 仮想サーバー。必要なときに立ち上げて使い、自由にカスタマイズ可能 | アプリケーションを動かすコンピュータ |
| S3<br>Simple Storage Service | オブジェクトストレージ。写真・動画・ログなどの大容量データ保存に適している | 大きな倉庫にデータをしまう |
| RDS<br>Relational Database Service | 管理されたデータベースサービス。データの検索や管理を簡単に行える | 情報を検索・取り出すデータベース |

以下にシンプルなイメージ図を示します。  

```plaintext
インターネット
    │
  VPC (Private Network)
    ├── EC2 (仮想サーバー)
    ├── RDS (データベース)
    └── S3 (ストレージ)
```

## 4. VPC( Virtual Private Cloud )とは？ - AWS上の自分専用プライベートネットワーク

VPCはAWS上で作る「自分専用のネットワーク空間」です。外部からのアクセスを制限し、安全な環境を作ります。  
この中でEC2やRDSなどを配置します。

- インターネットからの出入り口を管理する  
- サブネットなどでネットワークを分割できる  

※ セキュリティグループはVPCそのものではなく、EC2などに対して通信許可・拒否を設定する別機能です。  

## 5. 安全にAWSを使うためのポイント - 権限管理・通信制御・監視の基本対策

AWSのクラウドは便利ですが、正しい設定なしではリスクがあります。  

- IAM( Identity and Access Management )でユーザーや権限を細かく管理する  
- Security Group( セキュリティグループ )で不要なアクセスを遮断する  
- Access Key( アクセスキー )を厳重に管理する  
- CloudTrail( 操作ログ記録 )などで定期的にログやアクセス状況を確認する  

## 6. AWSリソースの料金はどう決まるのか？ - 無駄な稼働を防ぐコスト管理の重要な仕組み

AWSの料金は使った分だけ払う「従量課金制」です。  

| 項目 | 料金がかかるポイント |
|-------|-------------------|
| EC2    | 稼働時間( 時間単位 )と利用したスペック |
| S3     | 保存データ容量とデータ転送量 |
| RDS    | 稼働時間とストレージ容量、通信量 |

AWSは従量課金制のため、使わないリソースを動かし続けると無駄なコストが発生します。  
不要なリソースは停止または削除し、コストを抑えましょう。

## 7. インフラ基礎まとめ - AWSで安定した土台を作る第一歩

- インフラはシステムの土台  
- AWSは自前の設備なしでインフラを利用できるクラウドサービス  
- まずはEC2( 仮想サーバー )、S3( ストレージ )、RDS( データベース )の3サービスを理解すること  
- VPCでネットワークの安全性を確保し、IAMで権限管理する  
- 料金は使った分だけ。不要リソースは止めてコストを抑える  

## 8. 重要用語説明

| キーワード | 説明 |
|---|---|
| Infrastructure( インフラ ) | ITシステムを支えるネットワーク、サーバー、ストレージなどの基盤設備 |
| AWS<br>Amazon Web Services | Amazonが提供するクラウドサービス。必要なIT資源をインターネット経由で利用できる |
| EC2<br>Elastic Compute Cloud | AWSの仮想サーバーサービス。必要な台数・性能でサーバーを利用できる |
| S3<br>Simple Storage Service | AWSのストレージサービス。ファイルや画像、ログなどを保存できる |
| RDS<br>Relational Database Service | AWSのデータベースサービス。MySQLなどのDBを簡単に利用できる |
| VPC<br>Virtual Private Cloud | AWS上に作る専用ネットワーク空間。安全にシステムを構築できる |
| IAM<br>Identity and Access Management | AWSのユーザー・権限管理サービス。利用者ごとに操作権限を設定できる |
| Security Group( セキュリティグループ ) | AWSの通信制御機能。サーバーへの接続許可・拒否を設定できる |
| Access Key( アクセスキー ) | AWSへプログラム接続する際に使う認証情報。厳重な管理が必要 |
| CloudTrail | AWSの操作履歴記録サービス。誰が何をしたか確認できる |
