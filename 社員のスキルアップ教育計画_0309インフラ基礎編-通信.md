# クラウド通信<br>( AWSクラウドと社内ネットワークを安全につなぐ通信技術 )
企業のネットワークとAWSのクラウドを安全かつ効率的に利用するためには、  
「外部からどのようにアクセスするか」と「ネットワークをどのように接続するか」を設計する必要があります。  

本章では、踏み台サーバによる安全なアクセス方法と、  
VPN( Virtual Private Network )・拠点間VPN( Site-to-Site VPN )・Direct Connect( 専用線接続 ) によるネットワーク接続の仕組みを解説します。  

これらはインフラの「通り道」を作り、安定した通信を確保する重要な技術です。  
実務では、セキュリティ要件・通信速度・運用コストを考慮して適切に使い分けます。

👉 **「アクセス制御」と「ネットワーク接続」は別の設計要素です**

## 1. 接続の基本( IPアドレスとDNS名 )
サーバへ接続するときは、通信方式の前に **「何を指定して接続するか」** を理解することが重要です。  
Tera Term や SSH クライアントでは、通常以下のいずれかを接続先として指定します。

### 接続先として指定するもの
* **IPアドレス**
  * パブリックIP( インターネット公開用 )
  * プライベートIP( 社内NW・VPN接続用 )
* **DNS名( ホスト名 )**
  * EC2 の Public DNS
  * 社内DNSで名前解決した内部サーバ名

※PublicIP は外部から到達可能なIPアドレス、PrivateIP は社内ネットワーク内だけで利用されるIPアドレスです。

👉 **VPN・専用線・踏み台は「接続経路」であり、最終的に指定するのはサーバのIPアドレスやDNS名です。**

## 2. 踏み台サーバ( Bastion )による接続
外部から直接サーバーへアクセスさせず、**踏み台サーバを経由して内部を保護する構成**です。

### 特徴
- 公開するのは踏み台サーバのみ( 内部サーバは非公開 )
- Security Groupで通信を制御し、許可された通信のみ通す
- 内部サーバは踏み台サーバからのアクセスのみ許可

👉 **「公開は最小限・内部は閉じる」が基本設計**

### 接続イメージ
```plaintext
   端末
    │
インターネット
    │ 【接続手順1】公開されたEC2 に対してSSH接続
    ▼
  VPC( プライベートネットワーク )
    │
    └── EC2( 踏み台サーバ / Public )
           └─ Security Group ... リソース単位で設定するホワイトリスト型の通信制御ルール
                ・インバウンド: SSH(22) → 自分のIPのみ許可
           │
           │ 【接続手順2】内部サーバへSSH接続
           ▼
        EC2( 内部サーバ / Private )
           └─ Security Group ... リソース単位で設定するホワイトリスト型の通信制御ルール
                ・インバウンド: SSH(22) → 踏み台のSGのみ許可( インターネットからの直接アクセスは不可 )
```
- 接続手順1: 端末でTeraTerm① → 踏み台の **PublicIP** に対してSSH接続
- 接続手順2: 端末でTeraTerm② → 踏み台から( 同じTeraterm内で )内部サーバの **PrivateIP** へSSH接続

### 実務での使いどころ
* 開発者や運用担当者がサーバーへ安全にログインする場合  

👉 **リモート接続( 踏み台 )は「人のアクセス」**  

## 3. VPN( Virtual Private Network )とは？

インターネットを利用して、**安全な専用線のような通信を実現する接続方式**です。

### 特徴
* インターネット経由で接続するため導入コストが低い
* 通信は暗号化され、安全性を確保できる
* 環境に依存するため、通信速度や安定性にばらつきがある

### 接続イメージ
```plaintext
   端末
    │ 【接続手順1】VPN接続
インターネット
    │ │
  VPC( プライベートネットワーク )
    │ │
    ▼ │
  VPN接続後の端末
       │ 【接続手順2】内部サーバへSSH接続
       ▼
   EC2( 内部サーバ )
```
- 接続手順1: 端末でVPN接続 → あなたのPCが **社内/AWSネットワーク** に参加
- 接続手順2: 端末でTeraTerm → 内部サーバの **PrivateIP** へSSH接続

### 実務での使いどころ
* 低コストでクラウド接続を始めたい場合
* 拠点が複数でも柔軟に接続が必要な場合
* 短期間で構築・変更したい場合

👉 **「低コスト・柔軟性」を重視したネットワーク接続**

## 4. 拠点間VPN( Site-to-Site VPN )とは？
オンプレミスとAWSを**ネットワーク単位で接続**する方式です。
* 拠点全体をAWSと接続
* 個別サーバではなくネットワーク同士を接続するイメージ
* 複数拠点がある企業で利用される

### 接続イメージ
```plaintext
       端末
        │ 【接続手順】各サービスへそのまま接続可能
本社データセンター
     │ │
インターネット
     │【接続済】インターネットでVPNトンネル接続
     ▼ │
VPC( プライベートネットワーク )
        │
        ▼
   EC2( 内部サーバ )
```
- 接続手順: 端末でTeraTerm → 内部サーバの **PrivateIP** へSSH接続

### 実務での使いどころ
- 社内ネットワーク全体をAWSと接続したい場合
- 拠点間( 本社・支社 )をまとめてクラウドと接続する場合
- 個別サーバではなく、ネットワーク単位で一括接続したい場合

👉 **「サーバ単位」ではなく「ネットワーク単位」で接続するのが特徴**

## 5. Direct Connect( 専用線接続 )とは？

AWSと企業ネットワークを、**専用回線で直接接続する方式**です。

### 接続イメージ
```plaintext
       端末
        │ 【接続手順】各サービスへそのまま接続可能
本社データセンター
     │ │
Direct Connect
     │【接続済】専用回線で直接接続
     ▼ │
VPC( プライベートネットワーク )
        │
        ▼
   EC2( 内部サーバ )
```
- 接続手順: 端末でTeraTerm → 内部サーバの **PrivateIP** へSSH接続

### 特徴

* インターネットを経由しないため、高速かつ安定した通信が可能
* 通信の品質が高く、低遅延を実現できる
* 初期費用・運用コストが高く、導入に時間がかかる

### 実務での使いどころ
* 大規模基幹システム
* 大量データ転送
* 低遅延が必要なリアルタイム処理

👉 **「高速・安定性」を重視したネットワーク接続**

## 6. 比較表
👉 **通信方式の違いは「誰と何を、どう繋ぐか」で整理すると理解しやすい**

| 観点 | 踏み台サーバ | VPN | 拠点間VPN | Direct Connect |
|---|:---:|:---:|:---:|:---:|
| 接続対象 | 人のアクセス | 人のアクセス | 拠点ネットワーク | 社内ネットワーク |
| 接続単位 | 端末単位 | 端末単位<br>( 利用者PCからVPCに接続 ) | ネットワーク単位<br>( 拠点ルータ同士接続 ) | ネットワーク単位 |
| 接続経路 | インターネット | インターネット | インターネット(IPsec) | 専用回線 |
| 特徴 | 踏み台経由で内部保護 | 外出先から安全接続 | 拠点全体を接続 | 高速・安定通信 |
| 強み | セキュアなログイン | 低コスト・柔軟 | 一括接続 | 高品質・低遅延 |
| 主な用途 | サーバログイン | テレワーク・保守 | 本社/支社接続 | 基幹・大規模連携 |

```plaintext
踏み台:
　PC → Bastion → EC2
VPN:
　PC ==VPN==> EC2
Site-to-Site:
　PC(社内) → LAN ==VPN==> EC2
Direct Connect:
　PC → 社内LAN ==専用線==> EC2
```

## 7. 実務での選び方
👉 利用者端末から見ると、目的地は同じAWSでも途中の接続方式が異なります。
- サーバ保守ログイン → 踏み台サーバ
- 個人利用中心 → VPN
- 拠点全体接続 → Site-to-Site VPN
- 高品質通信 → Direct Connect

## 8. 重要用語説明
| キーワード | 説明 |
|---|---|
| 踏み台サーバ( Bastion ) | 外部から内部サーバへ安全にアクセスするための中継サーバ |
| VPN<br>Virtual Private Network | インターネット経由で安全な通信を行う技術 |
| Direct Connect | AWSとオンプレミス間を結ぶ専用回線サービス |
| Site-to-Site VPN | 拠点間をネットワーク単位で接続するVPN方式 |
| VPC<br>Virtual Private Cloud | AWS内の仮想ネットワーク環境 |
| Security Group | リソース単位で通信を制御するホワイトリスト型ファイアウォール |
| Encryption( 暗号化 ) | データを第三者が読めないように変換する技術 |
| Latency( 通信遅延 ) | 電子データの送受信にかかる時間 |
